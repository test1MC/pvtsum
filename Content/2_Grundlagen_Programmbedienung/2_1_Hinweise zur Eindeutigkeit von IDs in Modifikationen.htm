Hinweise zur Eindeutigkeit von IDs in Modifikationen

Im Rahmen des Szenariomanagements achtet darauf, dass alle Netzobjekte innerhalb des gesamten Projekts eindeutige IDs besitzen. Die Verwaltung findet in der Datenbankdatei statt. So ist sichergestellt, dass in verschiedenen Modifikationen nicht Netzobjekte mit der gleichen ID existieren.

Wenn Sie beispielsweise ein neues Netzobjekt einfügen, schlägt automatisch die nächste im gesamten Projekt freie Nummer als eindeutige ID vor. Aus dem gleichen Grund setzt beim additiven Netzlesen und beim Einlesen von Modifikationsdateien für numerische Schlüssel automatisch die Einstellung Konfliktvermeidung aktiv. So ermittelt automatisch die nächste freie ID pro Netzobjekttyp und wählt den entsprechenden Offset.

Es kann jedoch auch sinnvoll sein, diese Vorgaben zu ändern. Beispielsweise möchten Sie in mehreren Szenarien die gleiche Haltestelle mit unterschiedlicher Lage oder unterschiedlichen Eigenschaften vergleichen. In diesem Fall erzeugen Sie eine Haltestelle mit der gleichen ID in mehreren Modifikationen. Die Modifikationen schließen sich dann gegenseitig aus, Sie können aber in verschiedenen Szenarien verwendet werden. Wenn Sie die Szenarien vergleichen, können Sie die Attribute der Haltestellen (und der entsprechenden Linien) vergleichen, weil sie die gleiche ID besitzen.

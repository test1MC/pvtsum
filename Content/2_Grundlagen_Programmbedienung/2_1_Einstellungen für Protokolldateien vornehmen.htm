Einstellungen für Protokolldateien vornehmen

Wählen Sie im Menü Bearbeiten den Eintrag Benutzereinstellungen.

Das Fenster Benutzereinstellungen öffnet sich.

Wählen Sie im Navigator den Eintrag Dateien & Protokolle > Allgemein.

Nehmen Sie im Bereich Meldungsdatei (messages.txt) die gewünschten Einstellungen vor.

Element

Beschreibung

Keine Datei schreiben

Über die Option legen Sie fest, dass Fehler nicht in eine Meldungsdatei ausgegeben werden.

Neue Datei erzeugen

Über die Option legen Sie fest, dass Fehler in eine neue Meldungsdatei ausgegeben werden.

Hinweis

Falls beim Programmstart schon eine Meldungsdatei vorhanden ist, wird sie überschrieben. Wenn das nicht möglich ist, wird eine Datei Messages1.txt angelegt.

Datei fortführen, falls vorhanden, ansonsten neue Datei erzeugen

Über die Option legen Sie fest, dass Fehler in eine schon vorhandene Meldungsdatei ausgegeben werden.

Hinweis

Falls beim Programmstart schon eine Meldungsdatei vorhanden ist, wird diese fortgeführt. Ansonsten wird eine neue Datei Messages.txt angelegt.

Nehmen Sie im Bereich Protokolldatei (protocol.txt) die gewünschten Einstellungen vor.

Element

Beschreibung

Keine Datei schreiben

Über die Option legen Sie fest, dass keine Protokolldatei ausgegeben wird.

Neue Datei erzeugen

Über die Option legen Sie fest, dass die Informationen in eine neue Protokolldatei ausgegeben werden.

Hinweis

Falls beim Programmstart schon eine Protokolldatei vorhanden ist, wird sie überschrieben. Wenn das nicht möglich ist, wird eine Datei Protocol1.txt angelegt.

Datei fortführen, falls vorhanden, ansonsten neue Datei erzeugen

Über die Option legen Sie fest, dass die Informationen in eine schon vorhandene Protokolldatei ausgegeben werden.

Hinweis

Falls beim Programmstart schon eine Protokolldatei vorhanden ist, wird diese fortgeführt. Ansonsten wird eine neue Datei Protocol.txt angelegt.

Standardmäßig werden Meldungsdatei und Protokolldatei in dem Verzeichnis gespeichert, das in der Projektverzeichnisdatei für den Eintrag Log-Datei festgelegt ist (Speicherort von Dateien ändern). Diese Einstellungen können Sie ändern ( mit anderen Einstellungen starten).

Wählen Sie im Navigator den Eintrag Benutzeroberfläche > Warnungen.

Element

Beschreibung

Warnungen und Hinweise in Meldungsdatei schreiben

Über die Option legen Sie fest, dass zusätzlich zu den Fehlermeldungen auch Warnungen und Hinweise, die zum Beispiel beim Einlesen des Netzes oder von Nachfragedaten auftreten, in der Meldungsdatei gespeichert werden.

Warnungen während der Ausführung von Verfahren an der Oberfläche unterdrücken

Über die Option legen Sie fest, dass Warnungen, die den Verfahrensablauf nicht gefährden, nicht auf dem Bildschirm ausgegeben werden. Dies ist vor allem bei der iterativen Ausführung von Verfahren (mit Rücksprung) sinnvoll, da so der Ablauf nicht unnötig gestört wird.

Hinweis

Warnungen, die die Ausführung eines Verfahrens unmöglich machen, werden auch weiterhin auf dem Bildschirm ausgegeben. Alle Warnungen können Sie unabhängig von dieser Option im Meldungsfenster anzeigen lassen (Fenster Meldungen).

Tipp

Die unterdrückten Warnungen können Sie in der Meldungsdatei protokollieren, um sie nachträglich dort zu überprüfen.

Warnhinweis beim Speichern anzeigen

Über die Option legen Sie fest, ob beim Speichern einer Version unter dem gleichen Namen eine Sicherheitsabfrage angezeigt wird (Einstellungen zum Öffnen und Speichern von Dateien vornehmen).

Wählen Sie im Navigator den Eintrag Dateien & Protokolle > Aktivitätsprotokoll.

Element

Beschreibung

Stufe

Über die Auswahlliste legen Sie fest, ob Sie das Aktivitätsprotokoll speichern möchten und wie detailliert der Inhalt sein soll.

  • 0 - Kein Protokoll: gibt kein Aktivitätsprotokoll aus.
  • 1 - Top-Level-Aktivitäten: Alle Transaktionen, Berechnungen und Operationen, die neue Netze erzeugen, werden protokolliert.
  • 2 - alle Aktivitäten: Alle Top-Level-Aktivitäten und Aktionen werden protokolliert.
  • 3 - alle Aktivitäten detailliert: Alle Top-Level-Aktivitäten und Änderungen mit Attributen (alte und neue Werte) werden protokolliert.
  • 4 - alle Aktivitäten maximal detailliert: Alle Top-Level-Aktivitäten und Lösch-Aktionen mit allen alten Attributen und Werten werden protokolliert.

Benutzer

Der eingegebene Name wird im Aktivitätsprotokoll mit den Änderungen protokolliert.

Größe überwachen

Über die Option legen Sie fest, ob bei Überschreitung einer bestimmten Größe der Aktivitätsprotokolldatei, die Sie im Eingabefeld Größe festlegen, eine Sicherheitsabfrage ausgibt.

Hinweis

Mit Hilfe der Sicherheitsabfrage entscheiden Sie, ob Sie die Protokollierung abschalten oder fortsetzen.

Die Änderungen werden übernommen.

Die Einstellungen werden sofort wirksam, Sie müssen Visum nicht neu starten.

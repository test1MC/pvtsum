Einstellungen in der Konfigurationsdatei ändern

Öffnen Sie die Konfigurationsdatei in einem Texteditor.

Nach der Installation finden Sie die Standard-Konfigurationsdatei unter ...\Programme\PTV Vision\\exe\VISUM.cfg

Eintrag

Standard

Beschreibung

* -E

EmissionFile

emiss.dat

Emissionsfaktoren des Schweizer Umweltbundesamtes für Umweltwirkungsanalysen, die beim Programmstart eingelesen werden

Hinweis

Diese Einstellung sollten Sie nicht ändern!

* -F

MessagesFile

messages.txt

Meldungsdatei, in die Fehlermeldungen ausgegeben werden (Mit Protokolldateien arbeiten)

* -G

GParFile

std.gpa

Grafikparameter (binär), die beim Programmstart eingelesen werden (Grafikparameter des Netzfensters speichern und lesen)

* -H

VersionFile

 

Version, die beim Programmstart geöffnet wird (optional) (Versionsdatei öffnen)

* -0

StdListLayoutPath

 

Listen-Layout, das beim Programmstart eingelesen wird (optional) (Listenlayout anpassen)

* -P

PathFile

std.pfd

Projektverzeichnisse, die beim Programmstart eingelesen werden (Projektverzeichnisdatei öffnen)

* -R

RlsFile

rls.dat

Korrekturfaktoren für unterschiedliche Straßenoberflächentypen bei der Lärmberechnung, die beim Programmstart eingelesen werden

Hinweis

Diese Einstellung sollten Sie nicht ändern!

* -T

LogFile

protocol.txt

Protokolldatei, in die Informationen ausgegeben werden (Mit Protokolldateien arbeiten)

* -U

ProceduresettingsFile

 

Verfahrensparameter, die beim Programmstart eingelesen werden (optional) (Verfahrensparameter in einer Datei speichern und aus Datei einlesen)

Ändern Sie die gewünschten Verzeichnisse und Dateinamen.

Sie können den Dateinamen mit oder ohne Pfad eingeben. Wenn Sie einen Dateinamen ohne Pfad eingeben, sucht in dem Verzeichnis, das in der Projektverzeichnisdatei für die Dateiart angegeben ist (Projektverzeichnisdatei öffnen und speichern).

Speichern Sie die Datei bei Bedarf in einem anderen Verzeichnis und/oder unter einem anderen Namen.

Wir empfehlen, die geänderte Datei nicht im Programmverzeichnis zu speichern, damit sie bei einer Installation oder einem Update nicht überschrieben wird. Wenn Sie die Datei zum Beispiel zusammen mit Ihren Projektdaten speichern, können Sie alle Daten gemeinsam archivieren.

Die Konfigurationsdatei wird gespeichert. Wenn Sie sie unter anderem Namen oder in einem anderen Verzeichnis gespeichert haben, müssen Sie noch festlegen, dass sie beim Programmstart verwendet wird (Beim Programmstart eine andere Konfigurationsdatei verwenden).

Beispiel für eine benutzerdefinierte Konfigurationsdatei
* -E
EmissionFile      emiss.dat
* -F
MessagesFile         C:\Projects\Project_xy\ProtocolFiles\Messages		.TXT
* -G
GParFile          C:\Projects\Project_xy\ListLayout\Layout.gpa
* -H
VersionFile       C:\Projects\Project_xy\Version_B.VER
* -0
StdListLayoutPath C:\Projects\Project_xy\ListLayout\Layout.lla
* -P
PathFile          C:\Projects\Project_xy\aktdrv.pfd
* -R
RLSFile           rls.dat
* -T
LogFile         C:\Projects\Project_xy\ProtocolFiles\Protocol.txt
* -U
ProceduresettingsFile C:\Projects\Project_xy\ProcSettings\PrPar.par

Grundeinstellungen eines Projekts bearbeiten

Nachdem Sie ein Projekt für das Szenariomanagement angelegt haben, können Sie dessen Grundeinstellungen bearbeiten.

Wählen Sie im Menü Datei den Eintrag Szenariomanagement > Projekt öffnen.

Das Fenster Szenariomanagement-Projekt öffnen öffnet sich.

Wählen Sie das gewünschte Projekt *.vpdb/*.vpdbx.

Klicken Sie auf die Schaltfläche Öffnen.

Das Fenster Projekt bearbeiten öffnet sich.

Sie können ein Projekt auch in öffnen, indem Sie im Windows-Verzeichnis auf die gewünschte Projektdatenbank *.vpdb/*.vpdbx doppelklicken.

Wählen Sie das Register Grundeinstellungen.

Sie können auf allen Registern des Fensters Projekt bearbeiten Text mit der Tastenkombination Strg + C kopieren und mit der Tastenkombination Strg + V an anderer Stelle einfügen.

Element

Beschreibung

Projektname

Name des Projekts. Standardmäßig steht hier der Name, den Sie beim Anlegen des Projekts vergeben haben. Wenn Sie den Namen ändern, hat das keinen Einfluss auf die Namen von Projektdatei und Verzeichnis.

Basisversion

Über die Schaltfläche Basisversion bearbeiten öffnen und bearbeiten Sie die Daten der Versionsdatei (siehe „Basisversion bearbeiten“ auf Seite 1).

Projektverzeichnisse

Über die Schaltfläche Projektverzeichnisse bearbeiten legen Sie pro Dateityp fest, wie Sie die entsprechenden Daten verwalten möchten:

  • in einem gemeinsamen Verzeichnis (Ordner SharedData),
  • pro Szenario separat (Ordner Scenarios),

Sie können ihre Projektdaten strukturiert verwalten, wenn Sie in den Projektordnern Scenario und SharedData Unterordner anlegen. In der Spalte Ordner (relativ) können Sie über die Schaltfläche den Speicherort für jede Dateiart festlegen.

Wenn Sie im Verfahrensablauf eine entsprechende Datei speichern oder öffnen möchten, geben Sie nur den Dateinamen ein. Über die Projektverzeichnisse ist festgelegt, woher die Datei geholt und wohin sie geschrieben wird.

Hinweis

Wenn Sie die Basisversion bearbeiten, wird automatisch der Ordner SharedData als Speicherort vorgeschlagen.
Projekteinstellungen

Über die Schaltfläche öffnet sich ein neues Fenster, in dem Sie festlegen können, dass Sie Attribute mit Default-Werten nicht in Modifikationen speichern möchten.

Hinweis

Ist die Option ausgewählt, werden beim Erzeugen der Modelltransferdateien beim Bearbeiten einer Modifikation Attribute, die auf Standardwerten stehen, nicht gespeichert. Bei einem Umstieg auf die nächste Hauptversion ist die Bedeutung von Dateien mit einem abweichenden Default anders.

Logbuch

Über das Symbol  Logbucheintrag einfügen legen Sie eine neue Zeile am Ende des Logbuchs an. In diese können Sie einen Text eingeben und bei Bedarf das Datum ändern.

Szenario-Kennziffer

Über das Symbol  Szenario-Kennziffern auswählen wählen Sie Relationen aus (siehe „Fenster zur Auswahl mehrerer Attribute“ auf Seite 1 und „Indirekte Attribute: Relationstypen und Aggregationsfunktionen“ auf Seite 1). Diese Kennziffern stehen Ihnen nach der Berechnung von Szenarien im Register Szenarien zur Anzeige zur Verfügung (siehe „Szenarien auf dem lokalen Rechner berechnen“ auf Seite 1). Sie enthalten über das ganze Netz aggregierte Werte, über die Sie mehrere Szenarien miteinander vergleichen können.

Über das Symbol  Kurznamen für Szenario-Kennziffer bearbeiten ändern Sie den Namen, mit dem eine Szenario-Kennziffer in der Liste angezeigt wird.

Nehmen Sie in den weiteren Registern die gewünschten Einstellungen vor.

Element

Beschreibung

Szenarien

  • Szenarien anlegen und verwalten (siehe „Szenarien verwalten“ auf Seite 1)
  • Szenarien berechnen (siehe „Szenarien berechnen und Ergebnisse anzeigen“ auf Seite 1)
  • Szenarien vergleichen (siehe „Szenarien vergleichen“ auf Seite 1)
  • Szenarien exportieren (siehe „Szenario als Versionsdatei speichern“ auf Seite 1)

Modifikationen

  • Modifikationen anlegen und verwalten (siehe „Modifikationen verwalten“ auf Seite 1)
  • Modifikationen exportieren (siehe „Modifikation als Modelltransferdatei speichern“ auf Seite 1)

Verfahrensparametersätze

Verfahrensparametersätze verwalten (siehe „Verfahrensparameter festlegen“ auf Seite 1)

Vergleichsmuster

Vergleichsmuster für den Vergleich von Szenarien verwalten (siehe „Vergleichsmuster anlegen“ auf Seite 1)
Verteiltes Rechnen

Verteiltes Rechnen von Szenarien verwalten

(siehe „Rechenknoten verwalten“ auf Seite 1)

Mehrbenutzerbetrieb Mehrbenutzerbetrieb verwalten (siehe Mehrbenutzerbetrieb im Szenariomanagement auf Seite 1)

Klicken Sie auf die Schaltfläche Projekt schließen.

Das Projekt wird automatisch gespeichert und geschlossen.

Die Verzeichnisstruktur des Projekts im Windows-Explorer dürfen Sie nicht ändern.

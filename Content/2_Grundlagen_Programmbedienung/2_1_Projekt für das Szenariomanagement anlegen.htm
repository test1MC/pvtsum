Projekt für das Szenariomanagement anlegen

Wenn Sie mit dem Szenariomanagement arbeiten möchten, müssen Sie ein Projekt anlegen Siehe "Projekte richtig organisieren". Dabei wählen Sie eine Versionsdatei aus, die als Basis für das Projekt dienen soll. Sie enthält beispielsweise den Ist-Zustand als Grundlage für weitere Planungen.

Erstellen Sie eine Basisversion *.ver für Ihr Projekt.

Wählen Sie im Menü Datei den Eintrag Szenariomanagement > Neues Projekt.

Das Fenster Neues Projekt erzeugen öffnet sich.

Element

Beschreibung

Projektname

Name des Szenariomanagement-Projekts

Dieser Name wird für die Bezeichnung der Projektdatei und des Verzeichnisses verwendet. Später können Sie einen sprechenden Namen für das Projekt vergeben (Grundeinstellungen eines Projekts bearbeiten).

Basisverzeichnis

Verzeichnis, in dem das Projekt gespeichert wird

Über die Schaltfläche wählen Sie das Verzeichnis.
Basisversion

aktuell geladene Daten als Basisversion verwenden

Ist die Option ausgewählt, wird eine Kopie der aktuell geöffneten Versionsdatei als Basisversion des Projekts verwendet. Die Datei wird in das festgelegte Verzeichnis kopiert.

existierende Version als Basisversion verwenden

Ist die Option ausgewählt, wird eine Kopie einer Datei *.ver als Basisversion des Projekts verwendet. Die Datei wird in das festgelegte Verzeichnis kopiert.

Über das Symbol wählen Sie die Versionsdatei.
Projektdatenbank

Projektdatenbank in Datei verwalten

Ist die Option ausgewählt, wird das Projekt in einer SQL-CE-Datenbank gespeichert.

Hinweis

Diese Lösung ist nicht für den Mehrbenutzerbetrieb geeignet.

Projektdatenbank in SQL Server verwalten

Ist die Option ausgewählt, wird die Projektdatenbank in einem SQL Server gespeichert. Anschließend kann das Projekt von mehreren Nutzern bearbeitet werden.

Hinweis

Dies erfordert die vorherige Installation von SQL Server Express. SQL Server Express ist nicht Teil der Standard--Installation, sondern muss von Ihnen oder Ihrer IT-Abteilung installiert und konfiguriert werden. Für den Netzwerkbetrieb muss SQL Server Express im Netz erreichbar sein. SQL Server Express können Sie von der Microsoft-Webseite herunterladen. SQL Server Express sollte auf einem Rechner installiert werden, auf den alle Bearbeiter Zugriff haben.

Datenbankverbindung

Über diese Schaltfläche stellen Sie die gewünschte Datenbankverbindung ein. In einem weiteren Fenster können Sie folgende Einstellungen festlegen.

Server

Unter Servername wählen Sie den gewünschten Server aus.

Hinweis

Beim Öffnen des Fensters aktualisiert sich die Liste der Servernamen. Diese können Sie über die Schaltfläche Aktualisieren erneut laden. Alternativ können Sie einen Servernamen in das Feld eingeben.

Windows-Authentifizierung verwenden

Ist die Option nicht ausgewählt, können Sie Benutzername und Passwort für die Authentifizierung eingeben.

Datenbank

Hier legen Sie eine neue Datenbank an oder wählen eine existierende aus.

Verbindung testen

Über die Schaltfläche können Sie die Verbindung zur Datenbank vorab testen.

Im Basisverzeichnis wird ein neuer Ordner mit dem Projektnamen erstellt. Das Projekt wird darin als Datei *.vpdbx angelegt. Parallel dazu erstellt mehrere Unterverzeichnisse. Über die Datei *.vpdbx ist die angelegte Datenbank mit verknüpft. Das Fenster Projekt bearbeiten öffnet sich. Hier können Sie weitere Daten eingeben oder ändern (Grundeinstellungen eines Projekts bearbeiten).

Wenn Sie ein dateibasiertes Projekt anlegen, legt zusätzlich eine Datei *.vpdb an.

Die Verzeichnisstruktur des Projekts im Windows-Explorer dürfen Sie nicht ändern.

Vergleichsmuster anlegen

Als Grundlage für einen Vergleich zweier Szenarien legen Sie ein Vergleichsmuster an. Hier legen Sie fest, welche Netzobjekte und Attribute Sie konkret vergleichen möchten. Die Vorgehensweise entspricht der beim Versionsvergleich (Versionen vergleichen). Sie können beide Arten des Versionsvergleichs anwenden.

Stellen Sie sicher, dass das gewünschte Projekt geöffnet ist (Grundeinstellungen eines Projekts bearbeiten).

Wählen Sie im Fenster Projekt bearbeiten das Register Vergleichsmuster.

Klicken Sie auf das Symbol   Vergleichsmuster einfügen.

Alternativ können Sie über das Symbol  Vergleichsmuster duplizieren ein vorhandenes Vergleichsmuster kopieren.

Das Fenster Vergleichsmuster anlegen öffnet sich.

Element

Beschreibung

Vergleich definieren

auf Berechnungsergebnissen der Szenarien

Ist die Option ausgewählt, öffnen Sie für das Erstellen des Vergleichsmusters die Versionsdateien von zuvor berechneten Szenarien. Sie sollten diese Option wählen, wenn Sie beispielsweise Berechnungsergebnisse über Streckenbalken darstellen möchten.

auf Eingabedaten der Szenarien

Ist die Option ausgewählt, öffnen Sie für das Erstellen des Vergleichsmusters die Netze der Szenarien. Diese enthalten keine Berechnungsergebnisse. Diese Option eignet sich, wenn Sie die Netze von Szenarien vergleichen möchten.

Szenarien zur Definition des Vergleichs

Führendes Szenario

Wählen Sie das Szenario aus, das beim Erstellen des Vergleichsmusters als führende Versionsdatei verwendet wird. In dieser Versionsdatei werden die Vergleichsattribute angezeigt.

Vergleichsszenario

Wählen Sie das Szenario aus, das beim Erstellen des Vergleichsmusters als Vergleichs-Versionsdatei verwendet wird. Aus dieser Versionsdatei stammen die Vergleichsattribute, die in der führenden Versionsdatei angezeigt werden.

Hinweise

Wenn Sie die Option auf Berechnungsergebnissen der Szenarien gewählt haben, werden in diesen Auswahllisten nur berechnete Szenarien angeboten.

Welche Szenarien Sie hier auswählen, ist nicht entscheidend, da Sie beim Anwenden des Vergleichsmusters andere Szenarien wählen können. Sie sollten Szenarien wählen, die die Daten enthalten, die Sie vergleichen möchten, so dass Sie beispielsweise die Grafikparameter passend wählen können.

Vergleichsmuster für Versionsvergleich mit Übernahme der Attribute anlegen

Wenn Sie einen Versionsvergleich mit Übernahme der Attribute ins führende Netz ausführen möchten, gehen Sie folgendermaßen weiter vor:

Wählen Sie unter Art des Versionsvergleichs die Option Attribute aus Vergleichsnetz übernehmen.

Element

Beschreibung

Art des Versionsvergleichs

Attribute aus Vergleichsnetz übernehmen

Ist diese Option ausgewählt, übernimmt die Attribute aus dem Vergleichsnetz in das führende Netz.

( Grundlagen: Versionsdateien vergleichen )

Streckenbalken automatisch auf Differenzen umstellen

Ist die Option ausgewählt, werden Skalierungs- und Klassifizierungsattribute von Streckenbalken automatisch auf das durch den Versionsvergleich neu erzeugte Attribut mit der Differenz zwischen dem ursprünglichen und dem neu hinzugefügten Attributwert umgestellt.

Das Fenster Versionsvergleich erstellen (Benutzerdefinition) öffnet sich.

Schränken Sie bei Bedarf die Menge der zu vergleichenden Daten ein.

Element

Beschreibung

Berechnete Attribute ausblenden

Ist die Option ausgewählt, werden zusätzlich zu Schlüsselattributen nur ausgewählte lesbare Attribute und deren Werte verglichen.

Ist die Option nicht ausgewählt, werden sowohl lesbare als auch nicht lesbare Attribute und deren Werte verglichen.

Attribute mit Defaultbelegung ausblenden

Ist die Option ausgewählt, werden nur ausgewählte belegte Attribute und deren Werte verglichen.

Ist die Option nicht ausgewählt, werden alle ausgewählten Attribute und deren Werte verglichen, unabhängig davon, ob der Attributwert dem Standardwert entspricht oder davon abweicht.

Hinweis

Die Standardwerte der Attribute können Sie der Datei Attribute.xls unter ...\Programme\PTV Vision\\Doc entnehmen.

Leere Tabellen ausblenden

Ist die Option ausgewählt, werden nur Tabellen verglichen, die Netzobjekte enthalten.

Wählen Sie die Netzobjekttypen, die Sie vergleichen möchten.

Element

Beschreibung

Der Netzobjekttyp wird für den Vergleich verwendet.

Der Netzobjekttyp wird für den Vergleich nicht verwendet.

Alle Tabellen an

Über die Schaltfläche können Sie alle Netzobjekttypen auswählen.

Alle Tabellen aus

Über die Schaltfläche können Sie alle Netzobjekttypen und deren Attribute ausschalten.

Klicken Sie auf einen Netzobjekttyp.

Die Attribute und gegebenenfalls Subattribute des Netzobjekttyps werden angezeigt.

Sie können die Attribute mehrerer Netzobjekttypen gleichzeitig anzeigen, indem Sie die Taste Strg drücken, diese gedrückt halten und nacheinander auf die gewünschten Einträge klicken.

Abhängig von den Einstellungen im oberen Bereich des Fensters werden unter Umständen nicht alle Attribute eines Netzobjekttyps angezeigt.

Wählen Sie die Attribute und Subattribute, die Sie vergleichen möchten.

Element

Beschreibung

Das Attribut ist ausgewählt und wird verglichen.

Das Attribut ist nicht ausgewählt und wird nicht verglichen.

Das Attribut ist ein Schlüsselattribut und kann nicht ausgeschaltet werden.

Das Attribut ist ein Pflichtattribut.

Attributgruppen

Über die Schaltfläche können Sie vordefinierte Gruppen von Attributen in einem separaten Fenster auswählen oder ausschalten (Gruppen von Attributen für den Vergleich auswählen oder ausschalten).

Standard

Über die Schaltfläche blenden Sie leere Tabellen aus und berechnete Attribute ein. Die Auswahl der Attribute wird auf die Standardeinstellung zurückgesetzt.

Sie können mit der rechten Maustaste auf einen Netzobjekttyp klicken und über das Kontextmenü alle aktuell angezeigten Attribute des Netzobjekttyps aktivieren oder deaktivieren.

Alternativ können Sie die Auswahl der Netzobjekttypen und Attribute aus einer Netzdatei *.net einlesen (Layout aus einer Netzdatei einlesen).

Das Fenster Vergleichsmuster bearbeiten öffnet sich. Gleichzeitig öffnet sich das gewählte führende Szenario als Versionsdatei, in der Sie weitere Einstellungen vornehmen können.

Nehmen Sie im Fenster Vergleichsmuster bearbeiten die gewünschten Einstellungen vor.

Element

Beschreibung

Grafikparameter festlegen

Ist die Option ausgewählt, werden die in der aktuellen Versionsdatei verwendeten Grafikparameter mit dem Versionsmuster gespeichert.

Filter festlegen

Ist die Option ausgewählt, werden die in der aktuellen Versionsdatei verwendeten Filtereinstellungen mit dem Versionsmuster gespeichert.

Listen-Layouts festlegen

Ist die Option ausgewählt, werden die in der aktuellen Versionsdatei angezeigten Listen inklusive ihres Layouts mit dem Versionsmuster gespeichert.

Nehmen Sie bei Bedarf in der Versionsdatei die gewünschten Grafikparameter und Filtereinstellungen vor.

Die Einstellungen können Sie entweder über die entsprechenden Menüs festlegen oder einlesen.

Öffnen Sie bei Bedarf in der Versionsdatei die gewünschten Listen und legen Sie deren Listen-Layouts fest.

Sie können beispielsweise bestimmte Spalten einblenden oder ein vorhandenes Listen-Layout laden.

Klicken Sie auf die Schaltfläche Fertigstellen.

Im Register Vergleichsmuster wird der neue Eintrag angezeigt.

Vergleichsmuster für Versionsvergleich mit Vergleichsnetz im Hintergrund anlegen

Wenn Sie einen Versionsvergleich ausführen möchten, bei dem das Vergleichsnetz im Hintergrund geöffnet bleibt und die Attribute über Relationen verknüpft sind, gehen Sie nach Schritt 6. bitte folgendermaßen vor:

Wählen Sie unter Art des Versionsvergleichs die Option Vergleichsnetz im Hintergrund laden.

lädt das Vergleichsnetz im Hintergrund und verknüpft die Attribute über Relationen mit dem führenden Netz (Grundlagen: Versionsdateien vergleichen).

Nehmen Sie weitere Einstellungen vor.

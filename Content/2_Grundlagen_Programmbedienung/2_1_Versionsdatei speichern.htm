Versionsdatei speichern

Sie können eine Versionsdatei mit gleichem Pfad und Dateinamen speichern (d. h. die bestehende Versionsdatei wird überschrieben) oder die Versionsdatei unter neuem Dateinamen und/oder mit neuem Pfad speichern.

Eine Versionsdatei können Sie auch während des Verfahrensablaufs speichern.

Die Versionsdateien von sind nicht abwärts kompatibel. Sie können also Versionsdateien, die mit neueren -Versionen gespeichert wurden, nicht mit älteren -Versionen öffnen. Beispielsweise können Sie eine Versionsdatei von 13 nicht mit 12.50 öffnen. Die Nummer der Bugfix-Version, die mit einem Bindestrich an die Versionsnummer angehängt wird, ist davon nicht betroffen (Menü Hilfe > Info > Eintrag Version).
Versionsdatei mit gleichem Namen und Pfad speichern

Klicken Sie in der Symbolleiste Funktionen auf das Symbol  Version speichern.

Sie können die Version auch über das Menü Datei > Version speichern speichern oder indem Sie die Tastenkombination Strg+S drücken.

Wenn im Menü Bearbeiten > Benutzereinstellungen > Eintrag Benutzeroberfläche > Warnungen die Option Warnhinweis beim Speichern anzeigen ausgewählt ist, öffnet sich eine Sicherheitsabfrage (Einstellungen zum Öffnen und Speichern von Dateien vornehmen).

Bestätigen Sie die Sicherheitsabfrage gegebenenfalls mit Ja.

Die Version wird mit gleichem Pfad und gleichem Namen gespeichert und die bestehende Versionsdatei somit überschrieben.

Sie können auch automatisch eine Sicherungskopie speichern (Einstellungen zum Öffnen und Speichern von Dateien vornehmen).
Versionsdatei mit anderem Namen oder Pfad speichern

Wählen Sie im Menü Datei den Eintrag Version speichern unter.

Das Fenster Version speichern öffnet sich.

Wählen Sie bei Bedarf ein anderes Verzeichnis.

Ändern Sie bei Bedarf den Dateinamen.

Klicken Sie auf die Schaltfläche Speichern.

Die Versionsdatei wird gespeichert.

Versionsdatei während des Verfahrensablaufs speichern

Sie können die Versionsdatei während des Verfahrensablaufs speichern (Verfahrensablauf festlegen und starten).

Während des Verfahrensablaufs werden beim Überschreiben vorhandener Versionsdateien keine Sicherheitsabfragen angezeigt.

Klicken Sie in der Symbolleiste auf das Symbol  Verfahrensablauf öffnen.

Das Fenster Verfahrensablauf öffnet sich.

Fügen Sie an der gewünschten Stelle im Verfahrensablauf das Verfahren Version speichern aus der Kategorie Sonstiges ein (Verfahrensablauf festlegen und starten).

Stellen Sie sicher, dass das eingefügte Verfahren markiert ist, und klicken Sie im Bereich Aktionen auf Bearbeiten.

Das Fenster Parameter Version speichern öffnet sich.

Element

Beschreibung

Durchsuchen

Über die Schaltfläche öffnet sich ein Fenster, in dem Sie die Datei auswählen, in der die Version gespeichert werden soll.

Hinweis

Sie können sowohl eine vorhandene Datei auswählen als auch einen neuen Dateinamen eingeben. Wenn Sie eine vorhandene Datei wählen, müssen Sie eine Sicherheitsabfrage bestätigen.

Aktuellen Wert des innersten Rücksprung-Counters an Dateinamen anhängen

Ist die Option ausgewählt, wird an den gewählten Dateinamen ein Unterstrich und ein Zähler angehängt. Der Zähler enthält beim Speichern den Wert der aktuellen Iteration der innersten Rücksprungschleife (Zu einem Verfahren im Verfahrensablauf springen (Bedingter Rücksprung)).

Der Dateiname (ohne Zähler) wird in der Spalte Variante/Datei angezeigt.

Alternativ können Sie den Dateinamen ändern:

  • indem Sie auf den Eintrag in der Spalte Variante/Datei doppelklicken und ihn direkt bearbeiten
  • indem Sie auf das Symbol klicken und eine Datei auswählen.

Die Einstellung bezüglich des angehängten Zählers ändert sich dadurch nicht.

Starten Sie das Verfahren (Verfahrensablauf festlegen und starten).

Die Version wird während des Verfahrensablaufs zum festgelegten Zeitpunkt gespeichert.

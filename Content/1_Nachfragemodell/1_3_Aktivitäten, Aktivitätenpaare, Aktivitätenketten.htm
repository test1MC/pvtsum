Aktivitäten, Aktivitätenpaare, Aktivitätenketten

Das Nachfragemodell geht davon aus, dass Fahrtzwecke oder außerhäusliche Aktivitäten die Ursache von Mobilität sind. In den Beispielen dieser Dokumentation werden die Aktivitäten in der folgenden Tabelle benutzt. Sie sind aus Befragungen wie der KONTIV 89 (EMNID 1991) abgeleitet, wobei die Aktivität Ausbildung weiter differenziert wird.

Das Nachfrageobjekt Aktivität wird durch folgende Attribute beschrieben.

Attribut

Beschreibung

Code (Key)

Kurzbezeichnung (beliebiger String), zum Beispiel W

Name

Name der Aktivität, zum Beispiel Wohnen

IstHeimatAktivität

Dieses boolesche Attribut ist wahr (= 1), wenn die Aktivität den Ausgangs- und Endpunkt für eine Aktivitätenkette bildet. Typischerweise ist das für die Aktivität „Wohnen“ der Fall.

Nachfragemodell-Code

Kurzbezeichnung des Nachfragemodells, zu dem die Aktivität gehört (beliebiger String), zum Beispiel EVA-P

Aktivitäten sind optional und können nur für EVA- und VISEM-Modelle definiert werden. Bei Standard-Vier-Stufen-Modellen entspricht eine Aktivität genau einem Aktivitätenpaar.

Ein Aktivitätenpaar entspricht dem Weg zwischen zwei aufeinander folgenden Aktivitäten im Tagesablauf einer Person.

Das Nachfrageobjekt Aktivitätenpaar wird durch folgende Attribute beschrieben.

Attribut

Beschreibung

Code (Key)

Kurzbezeichnung (beliebiger String), zum Beispiel WA

Name

Name des Aktivitätenpaares, zum Beispiel Wohnen-Arbeiten

Nachfragemodell-Code

Kurzbezeichnung des Nachfragemodells, zu dem das Aktivitätenpaar gehört (beliebiger String), zum Beispiel DEFAULT

Folgende Attribute, die Aktivitätenpaare beschreiben, sind nur für EVA-Modelle relevant.

Attribut

Beschreibung

Quell-Aktivitäts-Code

Code der Aktivität, bei der der Weg beginnt, zum Beispiel W (Wohnen)

Ziel-Aktivitäts-Code

Code der Aktivität, bei der der Weg endet, zum Beispiel A (Arbeit)

Quelle-Ziel-Typ

Richtung des Aktivitätenpaares, bezogen auf die Heimataktivität

Folgende Werte sind möglich.

1 - Quellaktivität ist Heimataktivität (zum Beispiel Wohnen – Arbeit)

2 - Zielaktivität ist Heimataktivität (zum Beispiel Einkaufen – Wohnen)

3 - weder Quell- noch Zielaktivität ist Heimataktivität (zum Beispiel Sonstiges – Sonstiges)

Der Wert des Attributs wird standardmäßig aus dem Attribut IstHeimatAktivität von Quell- und Zielaktivität belegt, kann aber manuell überschrieben werden. Er beeinflusst die Berechnung in der Verkehrserzeugung und der Verkehrsverteilung (EVA-Verkehrserzeugung und EVA-Verkehrsverteilung und Moduswahl).

Eine Aktivitätenkette beschreibt eine Abfolge typisierter Aktivitätenpaare. So zum Beispiel die Kette Wohnung – Arbeit – Einkauf – Wohnung (WAEW). Eine solche Abfolge von Aktivitätenpaaren impliziert Ortsveränderungen, in diesem Beispiel drei Wege (WA, AE, EW).

Das Nachfrageobjekt Aktivitätenkette wird durch folgende Attribute beschrieben.

Attribut

Beschreibung

Code (Key)

Kurzbezeichnung (beliebiger String), zum Beispiel WAW

Name

Name der Aktivitätenkette, zum Beispiel Wohnen-Arbeit-Wohnen

Aktivitäten-Codes

Kommagetrennte Liste der Codes der zugehörigen Aktivitäten

Nachfragemodell-Code

Kurzbezeichnung des Nachfragemodells, zu dem die Aktivitätenkette gehört (beliebiger String), zum Beispiel DEFAULT

Im VISEM-Nachfragemodell wird das durchschnittliche Mobilitätsprogramm von Personen durch solche Aktivitätenketten beschrieben. Das Standard-Vier-Stufen-Modell und das EVA-Modell erlauben nur einelementige Aktivitätenketten, das heißt eine Aktivitätenkette entspricht 1:1 dem Aktivitätenpaar.

Anzahl nicht-konvergierter Netzobjekte in Protokolldatei ändern

Sie können für das Lernverfahren, das TRIBUT-Lernverfahren, die Stochastische Umlegung und die Dynamische Stochastische Umlegung festlegen, wie detailliert die Ausgabe der IV-Umlegungsergebnisse in die Protokolldatei sein soll. kann in der Protokolldatei eine Liste der bisher nicht konvergierten Netzobjekte oder Routen protokollieren. Sie legen fest, bis zu welcher Anzahl von Netzobjekten diese detaillierte Liste ausgegeben wird. Falls die Anzahl überschritten wird, werden die Netzobjekte zusammengefasst.

Damit die Ergebnisse ausgegeben werden, müssen Sie außerdem festlegen, dass eine Protokolldatei erstellt wird (Einstellungen für Protokolldateien vornehmen).

Wählen Sie im Menü Rechnen den Eintrag Allgemeine Verfahrenseinstellungen.

Das Fenster Allgemeine Verfahrenseinstellungen öffnet sich.

Wählen Sie im Navigator den Eintrag IV-Einstellungen.

Geben Sie die gewünschte Anzahl im Bereich Protokoll nicht konvergierter Objekte in das Eingabefeld maximale Anzahl nicht konvergierter Objekte, für die ein detailliertes Protokoll in der Protokolldatei erstellt wird ein.

Die Änderung wird übernommen. Sobald Sie eine IV-Umlegung durchführen, wird die neue Einstellung verwendet.

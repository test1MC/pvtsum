Modifikationen definieren

Wenn Sie Ihrem Projekt eine neue Modifikation hinzufügen, vergeben Sie zunächst einen eindeutigen Namen und ggf. eine Beschreibung dafür. Sie geben zudem an, von welchen bereits existierenden Modifikationen die neue Modifikation abhängt und ggf. mit welchen anderen Modifikationen sie sich ausschließt. lädt danach die Basisversion und alle zugrundeliegenden Modifikationen. Ein Schwebefenster informiert Sie darüber, dass alle Änderungen, die Sie nun am Modell vornehmen, zur Definition der neuen Modifikation zählen.

Sie können nun alle Bearbeitungsfunktionen auf dem Modell verwenden. Die Bearbeitung der Daten in der Projektansicht ist dagegen gesperrt. Wenn Sie alle Änderungen vorgenommen haben, beenden Sie die Definition, indem Sie im Schwebefenster auf die Schaltfläche Fertigstellen klicken. ermittelt alle Unterschiede zwischen dem aktuellen Modellzustand und demjenigen zu Beginn der Definition und speichert sie als Inhalt der neuen Modifikation.

Selbstverständlich können Sie in jedem Stadium Ihres Projekts weitere Modifikationen hinzufügen. Einmal angelegte Modifikationen lassen sich nachträglich verändern. Sie können sich zudem jederzeit den Inhalt der bestehenden Modifikationen anzeigen lassen.

Bei umfangreichen Projekten können Sie insbesondere die Erstellung der Modifikationen auf mehrere Bearbeiter verteilen. Voraussetzung dafür ist, dass die Projektdatenbank des Szenariomanagements in SQL Server verwaltet wird (siehe Benutzerhandbuch: Kap. 0.0.1 , Seite 1). Die von einzelnen Benutzern bearbeiteten Modifikationen sind dann für die Bearbeitung durch andere Benutzer gesperrt. Das gilt ebenso für davon abhängige Modifikationen oder Szenarien, die diese Modifikationen benutzen. Einen Überblick über gesperrte Objekte im Projekt können Sie sich im Register Mehrbenutzerbetrieb verschaffen. Durch die beschriebene Vorgehensweise ist sichergestellt, dass Szenarien und Modifikationen konsistent gehalten werden. Wollen Sie Ihr Projekt durch mehrere Benutzer bearbeiten, wird die Verwendung des Mehrbenutzerbetriebs empfohlen. Durch die angebotenen Konvertierungsfunktionen haben Sie jederzeit die Möglichkeit, das Projekt wieder in eine Standard SQL-CE Datenbank ohne Mehrbenutzerbetrieb zu konvertieren.

Sollte ein Mehrbenutzerbetrieb über das Netzwerk nicht möglich sein, können Sie alternativ die Bearbeitung manuell auf mehrere Benutzer verteilen. Gehen Sie dazu am besten folgendermaßen vor:

  • Geben Sie jedem Bearbeiter eine Kopie der Basisversion.
  • Jeder Bearbeiter bearbeitet seine Kopie der Basisversion, ohne das Szenariomanagement zu nutzen. Sobald er eine Modifikation fertig gestellt hat, speichert er die Änderungen gegenüber der Basisversion als Modelltransferdatei.
  • Lassen Sie sich alle dezentral erstellten Modelltransferdateien geben.
  • Öffnen Sie Ihr Projekt in . Für jede Modelltransferdatei führen Sie folgende Schritte aus:
  • Fügen Sie eine Modifikation in Ihr Programm ein.
  • Wenn Sie auffordert, die Modifikation ausgehend von der Basisversion zu modellieren, wenden Sie als einzige Aktion die Modelltransferdatei an.
  • Danach beenden Sie die Bearbeitung der Modifikation.
  • Definieren Sie auf Basis der Modifikationen Ihre Szenarien.
  • Mit diesem Ablauf erstellen mehrere Bearbeiter dezentral Modelltransferdateien, die Sie anschließend zentral unter die Szenarioverwaltung stellen.

Wenn mehrere Bearbeiter die Basisversion erweitern und Modelltransferdateien erzeugen, verwenden Sie unter Umständen die gleichen Schlüssel für neue Netzobjekte, obwohl diese inhaltlich Unterschiedliches bedeuten. erkennt solche Schlüsselkonflikte, wenn Sie die Modelltransferdateien unter Szenariomanagement-Verwaltung anwenden, und ersetzt die doppelt verwendeten Schlüssel automatisch durch eindeutige neue.

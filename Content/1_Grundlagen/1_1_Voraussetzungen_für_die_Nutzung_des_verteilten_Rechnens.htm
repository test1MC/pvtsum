Voraussetzungen für die Nutzung des verteilten Rechnens

Die für die verteilte Berechnung vorgesehenen Rechner (Rechenknoten) müssen bestimmte Voraussetzungen erfüllen:

  • Auf allen Rechnern müssen passende -Versionen installiert sein, die die gleiche Binärversion haben (Beispiel: 14.00-xx).
  • Auf allen Rechnern müssen die zur Berechnung erforderlichen Zusatzmodule, sowie eine ausreichende Netzgröße lizenziert sein.
  • Die verwendeten Lizenzen müssen das Modul "Verteiltes Rechnen" enthalten und der gleichen Lizenzgruppe zugeordnet sein.
  • Wenn Python-Skripte oder AddIns verwendet werden, muss auf allen Rechnern die passende Python-Version verfügbar und es müssen die verwendeten AddIns installiert sein.
  • Je nach Anwendungsfall muss sichergestellt werden, dass auf allen Rechnern zusätzliche Ressourcen wie beispielsweise die HBEFA-Datendateien oder benutzerdefinierte CR-Funktionen verfügbar sind. Daten, die spezifisch für das Projekt sind, werden von automatisch auf die beteiligten Rechner kopiert.
  • Gleichen Sie die Programmoptionen der Rechner ab, das geschieht nicht automatisch.
  • Die Rechner müssen gegenseitig über das Netzwerk erreichbar sein. Gegebenenfalls müssen Sie die für die Kommunikation verwendeten Ports in der Firewall der Rechner freigeben und entsprechende Benutzerrechte einräumen. Die Ports sind weitgehend frei wählbar. Damit die Rechenknoten automatisch von gefunden werden, müssen sie im gleichen Subnetz stehen. Ist dies nicht der Fall, können Sie sie trotzdem für das verteilte Rechnen nutzen, allerdings müssen Sie die Rechneradressen dann manuell eintragen.
  • Auf den Rechenknoten muss der zu der gewünschten Installation gehörende PTV Vision Berechnungs-Server laufen. Nur wenn diese Software ausgeführt wird, können Berechnungsaufträge entgegengenommen werden. Der Berechnungsserver wird bei der Installation von automatisch installiert. Es ist nicht erforderlich, dass selbst auf den Rechenknoten gestartet wird.
  • Der Berechnungsserver ist kein Windows-Dienst, die Software arbeitet nur, wenn ein Benutzer am Rechner angemeldet ist

Es wird eine Version von angeboten, die unabhängig von einer Vollversion installiert wird und ausschließlich für die verteilte Berechnungen von Szenarien und Verfahren verwendet wird.

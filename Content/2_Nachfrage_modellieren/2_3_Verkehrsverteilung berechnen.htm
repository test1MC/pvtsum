Verkehrsverteilung berechnen

Die Verkehrsverteilung ist die Bestimmung einer Gesamtnachfragematrix pro Nachfrageschicht aus bereits vorhandenen Quell- und Zielaufkommen der einzelnen Bezirke mittels relevanter Kenngrößen, zum Beispiel Reisezeiten, Fahrpreise (siehe Grundlagen: Kap. 0.0.0.1 , Seite 1).

Die Verkehrsverteilung wird mit dem Gravitationsmodell berechnet (siehe Grundlagen: Kap. 0.0.1 , Seite 1). Die Parameter für das Gravitationsmodell können Sie vorab schätzen (siehe „Gravitationsparameter schätzen (KALIBRI)“ auf Seite 1).

Stellen Sie sicher, dass das Quell- und Zielaufkommen der einzelnen Bezirke pro Nachfrageschicht als Bezirksattribute Quell- und Zielaufkommen vorliegen (siehe „Iterative Wiederholung vornehmen“ auf Seite 1 und „Aufkommen von Standard-4-Stufen-Nachfrageschichten ändern“ auf Seite 1).

Klicken Sie in der Symbolleiste auf das Symbol  Verfahrensablauf öffnen.

Das Fenster Verfahrensablauf öffnet sich.

Fügen Sie an der gewünschten Stelle im Verfahrensablauf das Verfahren Verkehrsverteilung aus der Kategorie Nachfragemodell ein (siehe „Verfahrensablauf festlegen und starten“ auf Seite 1).

Klicken Sie in der Zeile des neu eingefügten Verfahrens auf die Schaltfläche in der Spalte Bezugsobjekt(e).

Das Fenster Nachfrageschichten Auswahl öffnet sich.

Wählen Sie in der Auswahlliste Nachfragemodell auswählen das Nachfragemodell, zu dem die Nachfrageschichten gehören.

Wählen Sie in der Auswahlliste Nachfrageschicht die Nachfrageschichten, für die Sie die Verkehrsverteilung rechnen möchten.

Sie können mehrere Nachfrageschichten auswählen, indem Sie die Taste Strg drücken, diese gedrückt halten und auf die gewünschten Einträge klicken.

Die ausgewählten Nachfrageschichten werden in der Spalte Bezugsobjekt(e) angezeigt.

Stellen Sie sicher, dass das eingefügte Verfahren markiert ist, und klicken Sie im Bereich Aktionen auf die Schaltfläche Bearbeiten.

Das Fenster Parameter Verkehrsverteilung öffnet sich.

Nehmen Sie die gewünschten allgemeinen Einstellungen vor.

Element

Beschreibung

Nur für aktive Beziehungen

Ist die Option ausgewählt, werden nur die aktiv gesetzten Beziehungen berücksichtigt (siehe „Sonderfall: Beziehungsfilter“ auf Seite 1).

Beziehungen zwischen passiven Bezirken ausschließen

Ist die Option ausgewählt, werden nur die Beziehungen zwischen aktiv gesetzten Bezirken berücksichtigt (siehe „Netzobjekte über Filter aktiv und passiv setzen“ auf Seite 1 und „Netzobjekte über die räumliche Auswahl aktiv und passiv setzen“ auf Seite 1).

Alle Ergebnis-Nachfragematrizen vor der Berechnung auf 0 setzen

Ist die Option ausgewählt, werden alle den Nachfrageschichten zugeordneten Nachfragematrizen vor der Addition der berechneten Nachfrage auf 0 gesetzt.

Hinweis

Sie sollten die Option auswählen, wenn alle Nachfrageschichten, die derselben Nachfragematrix zugeordnet sind, in die Verkehrsverteilung eingehen.

Geschätzte Parameter nutzen

Ist die Option ausgewählt, werden die Parameter a, b, c und der Funktionstyp aus den Attributen der Nachfrageschicht verwendet.

Ist die Option nicht ausgewählt, legen Sie die Parameter und den Funktionstyp pro Nachfrageschicht in der Tabelle der Nachfrageschichten fest.

Für alle NSchichten übernehmen

Über die Schaltfläche übernehmen Sie von der markierten Nachfrageschicht die aktuellen Einstellungen der Kombination von Matrizen und Attributsvektoren und der Nutzenfunktion für alle anderen Nachfrageschichten.

Verteilungsmatrix

Verweis auf die Nachfragematrix, in die das Ergebnis der Verkehrsverteilung für die ausgewählte Nachfrageschicht gespeichert wird (siehe „Attribute und Optionen von Nachfrageschichten“ auf Seite 1. und „Matrizen auswählen“ auf Seite 1)

Funktionstyp

Über die Auswahlliste wählen Sie den Typ der Nutzenfunktion für die Berechnung der Verkehrsverteilung.

Hinweis

Wenn Sie die Option Geschätzte Parameter nutzen ausgewählt haben, ist die Option Logit eingestellt.

Tipp

Alternativ können Sie den Funktionstyp im Register Einstellungen im Fenster Wahlmodell für <Nachfrageschicht> festlegen. Das Fenster öffnet sich über die Schaltfläche (Quickinfo: Weitere Parameter).

Parameter

a, b, c

Geschätzte Parameterwerte für die Nutzenfunktion, die die Reaktion der Verkehrsteilnehmer auf Entfernungs- oder Zeitverhältnisse abbilden. Die Werte können auch negativ sein.

Hinweise

Wenn Sie die Option Geschätzte Parameter nutzen ausgewählt haben, sind die Felder nicht aktiv.

Wenn die Ausgangsmatrix Widerstände enthält, beispielsweise Entfernungen oder Reisezeiten, müssen Sie negative Exponenten für die Nutzenfunktion eingeben.

Tipp

Alternativ können Sie die Werte für die Parameter im Register Einstellungen im Fenster Wahlmodell für <Nachfrageschicht> festlegen. Das Fenster öffnet sich über die Schaltfläche (Quickinfo: Weitere Parameter).

Ausrichtungsparameter

Über die Option wählen Sie das Verteilungsmodell, das gerechnet werden soll.

gekoppelt bzgl. Quellaufkommen

Ist die Option ausgewählt, wird die Verkehrsverteilung einseitig gekoppelt bezüglich des Quellaufkommens berechnet.

Der Wert des Bezirksattributs Quellaufkommen wird für jeden Bezirk berücksichtigt.

gekoppelt bzgl. Zielaufkommen

Ist die Option ausgewählt, wird die Verkehrsverteilung einseitig gekoppelt bezüglich des Zielaufkommens berechnet.

Der Wert des Bezirksattributs Zielaufkommen wird für jeden Bezirk berücksichtigt.

Tipp

Alternativ können Sie die Werte für die Parameter im Register Einstellungen im Fenster Wahlmodell für <Nachfrageschicht> festlegen. Das Fenster öffnet sich über die Schaltfläche (Quickinfo: Weitere Parameter).

Verteilungsmatrix

Verweis auf die Nachfragematrix, in die das Ergebnis der Verkehrsverteilung für die ausgewählte Nachfrageschicht gespeichert wird (siehe „Attribute und Optionen von Nachfrageschichten“ auf Seite 1).

Klicken Sie auf die Schaltfläche (Quickinfo: weitere Parameter).

Das Fenster Wahlmodell für <Nachfrageschicht> öffnet sich.

Über die Option legen Sie fest, mit welchen Summenwerten das Multi-Verfahren gerechnet wird, wenn die Quell- und die Zielsumme unterschiedlich sind (siehe Grundlagen: Kap. 0.0.1 , Seite 1).

Hinweis

Klicken Sie auf die Schaltfläche Nutzendefinition  

Das Fenster Nutzendefinition für Nachfrageschicht <> öffnet sich.

Die Nutzendefinition ist eine Linearkombination von Matrizen, Attributsvektoren und/oder Konstanten, die die Erreichbarkeit der einzelnen Bezirke beschreibt (siehe „Matrizen und Attributsvektoren im Matrixfenster kombinieren“ auf Seite 1). Sie können pro Nachfrageschicht eine andere Nutzendefinition festlegen, mit der die Werte, die sich aus der Linearkombination ergeben, transformiert werden.

Das Fenster Matrix auswählen öffnet sich.

Wählen Sie die Nachfragematrix, in die das Ergebnis der Verkehrsverteilung für die ausgewählte Nachfrageschicht gespeichert wird. Dafür stehen Ihnen zwei Methoden zur Verfügung (siehe „Matrizen auswählen“ auf Seite 1)

Element Beschreibung

Matrix auswählen nach Nummer

Wählen Sie aus der Liste aller Matrizen anhand der Nummer die gewünschte Matrix aus.

Matrix auswählen nach Eigenschaften

Wählen Sie anhand bestimmter Attribute und deren Werte die Matrix aus.

Die ausgewählte Matrix wird übernommen und in der zweiten Spalte Matrix als Verweis angezeigt.

Führen Sie das Verfahren aus (siehe „Verfahren im Verfahrensablauf ausführen“ auf Seite 1).

Die Verkehrsverteilung wird berechnet, die Berechnungsergebnisse werden für jede Nachfrageschicht in die zugeordnete Ergebnismatrix gespeichert.

Sie können die Ergebnismatrizen anzeigen und ändern (siehe „Matrizen verwalten, anzeigen und analysieren“ auf Seite 1).

EVA-Bewertung berechnen

Vor der Verteilung der in der Stufe EVA-Verkehrserzeugung berechneten Quell- und Zielaufkommen der Bezirke ist eine Bewertung der Beziehungen nötig. Für die Bewertung werden so genannte Aufwandsarten definiert. Aufwandsarten sind Anteile der Gesamtkosten einer Fahrt, zum Beispiel Reisezeit, Zu- und Abgangszeit, monetäre Kosten, Anzahl Umsteigevorgänge im ÖV usw.

Bei der Bewertung werden Kenngrößenmatrizen je Modus, je Nachfrageschicht und je Aufwandsart mit Hilfe einer Bewertungsfunktion transformiert und anschließend multipliziert. Ausnahme: Für die Aufwandsart Reisezeit wird die transformierte Fußwegzeit (so genannte konkurrierende Fußwegzeit) – multipliziert mit einem Faktor – vom Ergebnis der Transformation subtrahiert. Die Ergebnisse werden je Nachfrageschicht in einer Kenngrößenmatrix mit dem Code EVA-Bewertung zusammengefasst (Grundlagen: EVA-Verkehrsverteilung und Moduswahl).

Wählen Sie in der Auswahlliste Nachfrageschicht die Nachfrageschichten, für die Sie die EVA-Bewertung rechnen möchten.

Sie können mehrere Nachfrageschichten auswählen, indem Sie die Taste Strg drücken, diese gedrückt halten und auf die gewünschten Einträge klicken.

Die ausgewählten Nachfrageschichten werden in die Spalte Bezugsobjekt(e) übernommen.

Stellen Sie sicher, dass das eingefügte Verfahren markiert ist, und klicken Sie im Bereich Aktionen auf die Schaltfläche Bearbeiten.

Das Fenster Parameter EVA-Bewertung öffnet sich.

Element

Beschreibung

Nur für aktive Beziehungen

Ist die Option ausgewählt, wird die EVA-Bewertung nur für aktive Beziehungen berechnet (Sonderfall: Beziehungsfilter).

Beziehungen zwischen passiven Bezirken ausschließen

Ist die Option ausgewählt, werden nur Beziehungen berücksichtigt, die mindestens einen aktiven Bezirk enthalten.

Alle Ergebnis-Matrizen vor der Berechnung auf 0 setzen

Ist die Option ausgewählt, werden alle Matrixwerte der Matrizen vor der Berechnung auf null gesetzt.

Heimatbezirk für Nachfrageschichten vom Typ 3 berücksichtigen

Ist die Option ausgewählt, wird der Heimatbezirk der Verkehrsteilnehmer für Nachfrageschichten mit Quelle-Ziel-Typ 3 als Potential berücksichtigt.

Ist die Option nicht ausgewählt, wird der Heimatbezirk ignoriert.

Ergebnis-Matrizen auf Datei schreiben

Ist die Option ausgewählt, werden die im Verfahren berechneten Kenngrößenmatrizen automatisch in der Versionsdatei gespeichert.

Hinweis

Standardmäßig ist die Option nicht ausgewählt. Bei Versionsdateien aus älteren Programmversionen wird die Option automatisch aktiviert.

Maximale Anzahl Relationen mit negativer Bewertung in der Protokolldatei

Dieser Wert bestimmt, wie detailliert die Daten in die Protokolldatei geschrieben werden. Wenn die Anzahl der Relationen mit negativer Bewertung geringer ist als der eingegebene Wert, werden die Relationen einzeln aufgelistet. Ist sie höher, werden die Relationen zusammengefasst.

Hinweis

Damit die Ergebnisse ausgegeben werden, müssen Sie außerdem festlegen, dass eine Protokolldatei erstellt wird (Einstellungen für Protokolldateien vornehmen).

Über die Schaltfläche können Sie ein numerisches Bezirksattribut wählen, das die Verfügbarkeit des ausgewählten Modus im jeweiligen Bezirk definiert.

Der Wert des Attributs sollte zwischen 0 und 1 liegen, wobei 0 nicht verfügbar und 1 zu 100 % verfügbar bedeutet.

Kapazitätsauslastung

Ob das Attribut auf Quell- und/oder Zielseite wirkt, hängt vom Quelle-Ziel-Typ des jeweiligen Aktivitätenpaars ab (Attribute und Optionen von Aktivitätenpaaren).

  • Typ 1 auf Zielseite
  • Typ 2 auf Quellseite
  • Typ 3 auf beide Seiten

Tipp

Der Wert des Attributs sollte zwischen 0 und 1 liegen, wobei 0 keine Kapazitätsauslastung und 1 eine Kapazitätsauslastung von 100 % bedeutet.

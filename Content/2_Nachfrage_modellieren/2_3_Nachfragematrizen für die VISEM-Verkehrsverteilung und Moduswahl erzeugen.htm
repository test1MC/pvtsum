Nachfragematrizen für die VISEM-Verkehrsverteilung und Moduswahl erzeugen

Sind für Personengruppen noch keine Nachfragematrizen ausgewählt, in die das Ergebnis der VISEM-Verkehrsverteilung und Moduswahl pro Personengruppe gespeichert wird, können Sie neue Nachfragematrizen erzeugen und den Personengruppen für die Verkehrsverteilung und Moduswahl automatisch zuordnen.

Wählen Sie im Menü Nachfrage den Eintrag Nachfragemodelle.

Das Fenster Nachfragemodelle öffnet sich.

Wählen Sie das Register Basis.

Wählen Sie das gewünschte Nachfragemodell.

Klicken Sie auf die Schaltfläche Verfahrensablauf erzeugen.

Das Fenster Verfahrensablauf erzeugen öffnet sich.

Klicken Sie auf die Schaltfläche Nachfragematrizen.

Das Fenster Optionen zur Erzeugung von Nachfragematrizen öffnet sich.

Element

Beschreibung

Verteilungsmatrizen

keine Verteilungsmatrizen erzeugen

Ist die Option ausgewählt, erzeugt keine Verteilungsmatrizen.

Verteilungsmatrizen erzeugen (eine Matrix je Personengruppe)

Ist die Option ausgewählt, erzeugt je Personengruppe eine Verteilungsmatrix.

Moduswahlmatrizen

eine Matrix je Modus erzeugen

Ist die Option ausgewählt, erzeugt je Modus des Nachfragemodells eine eigene Matrix.

eine Matrix je Modus erzeugen und zusätzlich eine Matrix je Modus und Personengruppe erzeugen

Ist die Option ausgewählt, erzeugt je Modus eine eigene Matrix und zusätzlich je eine Matrix für jede Kombination aus Modus und Personengruppe.

Aktivitätsmatrizen

keine Aktivitätsmatrix erzeugen

Ist diese Option ausgewählt, erzeugt keine Aktivitätsmatrix.

eine Matrix je Zielaktivität erzeugen

Ist diese Option ausgewählt, erzeugt für jede Zielaktivität eine Matrix.

eine Matrix je Zielaktivität erzeugen und zusätzlich eine Matrix je Zielaktivität und Personengruppe erzeugen

Ist diese Option ausgewählt, erzeugt für jede Zielaktivität eine Matrix und zusätzliche je eine Matrix für jede Kombination aus Zielaktivität und Personengruppe.

Sie erhalten zu jeder Matrizenart eine Information darüber, wie viele Matrizen aufgrund Ihrer Einstellungen erzeugt werden.

Für alle Objekte, denen noch keine Nachfragematrix zugeordnet ist, werden neue Nachfragematrizen erzeugt und jeweils zugeordnet.

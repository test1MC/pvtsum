Mehrstufige Moduswahl berechnen (Nested Mode Choice)

Alternativ zur einstufigen Moduswahl können Sie auch ein verschachteltes Moduswahl-Modell berechnen, für den Sie einen Entscheidungsbaum mit Modi an den Blättern und Nestern an den inneren Knoten konstruieren. Standardmäßig ermittelt die sich für die Nester ergebenden Nachfragematrizen automatisch, wobei der Nutzen eines Nests der LogSumme der Nutzen seiner Unterknoten entspricht. Diese temporären Matrizen können Sie für die Überprüfung der Nestmatrizen oder zur Parametrisierung eines individuellen nicht-sequentiellen Wahlmodells überschreiben (Grundlagen: Mehrstufige Moduswahl (Nested Mode Choice)).

Klicken Sie in der Symbolleiste auf das Symbol  Verfahrensablauf öffnen.

Das Fenster Verfahrensablauf öffnet sich.

Fügen Sie an der gewünschten Stelle im Verfahrensablauf das Verfahren Mehrstufige Moduswahl aus der Kategorie Nachfragemodell ein (Verfahrensablauf festlegen und starten).

Klicken Sie in der Zeile des neu eingefügten Verfahrens auf die Schaltfläche in der Spalte Bezugsobjekt(e).

Das Fenster Nachfrageschichten Auswahl öffnet sich.

Wählen Sie in der Auswahlliste Nachfragemodell auswählen das Nachfragemodell, zu dem die Nachfrageschichten gehören.

Wählen Sie in der Auswahlliste Nachfrageschicht die Nachfrageschichten, für die Sie die mehrstufige Moduswahl rechnen möchten.

Sie können mehrere Nachfrageschichten auswählen, indem Sie die Taste Strg drücken, diese gedrückt halten und auf die gewünschten Einträge klicken.

Die ausgewählten Nachfrageschichten werden in die Spalte Bezugsobjekt(e) übernommen.

Stellen Sie sicher, dass das eingefügte Verfahren markiert ist, und klicken Sie im Bereich Aktionen auf die Schaltfläche Bearbeiten.

Das Fenster Parameter Nested Mode Choice öffnet sich.

Beschreibung

Ist die Option ausgewählt, werden alle den Nachfrageschichten zugeordneten Nachfragematrizen vor der Addition der berechneten Nachfrage auf 0 gesetzt.

Hinweis

Sie sollten die Option auswählen, wenn alle Paare aus Nachfrageschicht und Modus, die derselben Nachfragematrix zugeordnet sind, gemeinsam in die Moduswahl eingehen.

Nachfrageschicht

Über die Auswahlliste können Sie die Nachfrageschicht je Nest oder Blatt wählen, für die die Einstellungen gelten sollen.

Moduswahlmodell

Über die Schaltfläche können Sie eine Nutzenfunktion in einem separaten Fenster einstellen, mit der die Werte, die sich aus der Kombination ergeben, transformiert werden (Parameter der Nutzenfunktion für die mehrstufige Moduswahl einstellen).

markiertes Nest/markierter Modus

Anzeige des im Entscheidungsbaum aktuell markierten Nests oder Knoten

Input-Nachfragematrix

Verweis auf die Nachfragematrix, die in der jeweiligen Entscheidungsstufe auf die zugehörigen Unterknoten aufgeteilt wird.

Hinweis

Die Input-Matrix entspricht der Ergebnismatrix des Vaterknotens.

Ergebnis-Nachfragematrix

Verweis auf die Nachfragematrix, die in der jeweiligen Entscheidungsstufe je Entscheidungsbaumeintrag gespeichert wird.

Matrixauswahl

Über die Schaltfläche können Sie für das markierte Nest eine Ergebnismatrix in einem separaten Fenster auswählen (Fenster Matrizen auswählen).

Automatische Nutzenskalierung für Unterknoten

Ist für ein Nest der Koeffizient der LogSumme nicht 1 oder 0, müssen die Koeffizienten der zugehörigen Unterknoten (Spalte Koeffizient) mit dem Quotienten 1/LogSummenkoeffizient skaliert werden.

Über die Option stellen Sie ein, ob die Koeffizienten der Unterknoten automatisch skaliert werden sollen oder nicht.

Hinweise

Die Option ist nur für Nester verfügbar.

Wenn der LogSummenkoeffizient 0 ist, werden keine Koeffizienten der Unterknoten verändert.

Nutzen U

Linearkombination von Matrizen, Attributsvektoren und/oder Konstanten je Modus oder Nest (Matrizen und Attributsvektoren im Matrixfenster kombinieren)

Bei Nestern beinhaltet die Nutzenfunktion außerdem die LogSumme über alle Unterknoten, die Sie mit einem beliebigen Koeffizienten glätten können, wenn als Moduswahlmodell eine Logit-Funktion eingestellt ist (Grundlagen: Mehrstufige Moduswahl (Nested Mode Choice)).

Hinweis

Das Ergebnis der Linearkombination wird als Nutzenmatrix interpretiert.

Matrix / Attribut /Konstante

Die Werte des gewählten Objekts können Sie mit einer Funktion transformieren (Spalte Transformation). Außerdem können Sie sie mit einem Faktor multiplizieren (Spalte Koeffizient).

Transformation

Über die Auswahlliste können Sie eine Funktion wählen, mit der die in der Spalte Matrix / Attribut / Konstante gewählten Werte umgerechnet werden (Matrixwerte bearbeiten).

Koeffizient

Faktor, mit dem die Werte elementweise multipliziert werden

Nutzenmatrix speichern in

Ist die Option ausgewählt, wird der berechnete Nutzen für spätere Auswertungen in einer Kenngrößenmatrix gespeichert, die Sie in der Auswahlliste wählen können.

Für alle NSchichten übernehmen

Über die Schaltfläche können Sie die Einstellungen der aktuell ausgewählten Nachfrageschicht bezüglich der Kombination von Matrizen und Attributsvektoren und der Nutzenfunktion für alle Nachfrageschichten des Nachfragemodells übernehmen.

Führen Sie das Verfahren aus (Verfahren im Verfahrensablauf ausführen).

Die mehrstufige Moduswahl wird berechnet. Die Ergebnisse werden für jedes Paar aus Nachfrageschicht und Modus in der zugeordneten Ergebnismatrix gespeichert.

Sie können die Ergebnismatrizen anzeigen und ändern (Matrizen verwalten, anzeigen und analysieren).

Element

Funktionstyp

Parameter

a, b, c

Geschätzte Parameterwerte für die Nutzenfunktion, die die Reaktion der Verkehrsteilnehmer auf Entfernungs- oder Zeitverhältnisse abbilden. Die Werte können auch negativ sein.

Hinweise

Wählen Sie das Register Funktionsgraph.

Der Kurvenverlauf der Nutzenfunktion wird mit den aktuell eingestellten Parametern a, b und c angezeigt.

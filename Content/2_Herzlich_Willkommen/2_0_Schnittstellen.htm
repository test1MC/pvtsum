Schnittstellen

Die nachfolgende Tabelle gibt einen Überblick über die Schnittstellen für den Datenimport und -export.

ANM-Schnittstelle (Vissim)

Beim Export wird aus dem -Netz ein abstrahiertes Netzmodell (ANM) erzeugt, das anschließend in Vissim eingelesen und bearbeitet werden kann (ANM-Schnittstelle). Änderungen im -Netz können auch später zu einem vorher exportierten und in Vissim veränderten Netz zugespielt werden. Das Vissim-Netz wird dabei nur so verändert, wie es für die Abbildung von -Änderungen unvermeidlich ist.

Die Funktion ist nur mit dem Zusatzmodul Vissim Export verfügbar.

Ferner können Sie Dateien *.anm in einlesen und die eingelesenen Daten bearbeiten (Abstrahiertes Netzmodell einlesen).

Vissig-Schnittstelle

Vissig ist ein externes Programm, das Ihnen die Verwaltung komplexer Signalisierungsdaten ermöglicht. Vorhandene Signalisierungsdaten können in externe Steuerungen konvertiert werden, in Vissig bearbeitet werden und anschließend wieder in eingelesen werden (LSA-Steuerung in Vissig bearbeiten).

Die Funktion ist nur mit dem Zusatzmodul Vissig verfügbar.

RBC-Steuerungen

RBC-Steuerungen ermöglichen Ihnen die Verwaltung komplexer Signalisierungsdaten (LSA-Steuerung vom Typ RBC bearbeiten).

Die Funktion ist nur mit dem Zusatzmodul RBC-Steuerungen verfügbar.

HAFAS-Import

Die HAFAS-Schnittstelle ermöglicht es, Datenbestände im HAFAS-Rohdatenformat in einzulesen (HAFAS-Import). Der Vorgang entspricht einem (nicht-additiven) Netzlesen, das heißt die Daten werden in ein anfänglich leeres Netz eingelesen.

Die Funktion ist nur mit dem Zusatzmodul HAFAS Import verfügbar.

railML®-Schnittstelle

Die railML®-Schnittstelle ermöglicht es, die für -Modelle relevanten Inhalte (insbesondere Fahrplan) aus dem railML® 2.0/2.1-Rohdatenformat (http://www.railml.org) einzulesen. Die Daten können additiv zu einem bestehenden Netz hinzugelesen werden, wobei Zuordnungen der Dateninhalte über Mapping-Attribute erfolgen (RailML®-Schnittstelle). Ferner können Sie Ihre -Daten nach railML® exportieren.

Die Funktionen sind nur mit den Zusatzmodulen RailML Import bzw. RailML Export verfügbar.

ÖV-Angebot aus Visum

Die ÖV-Angebot-Schnittstelle ermöglicht es Ihnen das aktuelle Netz durch ÖV-Angebotsdaten aus einem anderen -Netz zu aktualisieren (ÖV-Angebotsdaten Schnittstelle).

EMME/2-Import

EMME/2 ist eine Software zur Nachfragemodellierung (EMME/2-Import). hat eine Schnittstelle zu diesem System integriert, die Ihnen die Migration von EMME/2 Modellen nach ermöglicht.

TModel-Import

bietet Ihnen eine Schnittstelle für den Import von TModel-Daten (TModel-Import).
Synchro-Import bietet Ihnen eine Schnittstelle für den Import von Synchro-Daten (Synchro-Import).

SATURN-Import

Sie können Netzdaten in einlesen, die mit SATURN erzeugt wurden (SATURN-Import).
OpenStreetMap-Schnittstelle Über die Schnittstelle können Sie OSM-Daten in ein -Netz importieren (OpenStreetMap-Import).
-Schnittstelle Über die Schnittstelle können Sie Daten des ÖV zwischen den - und den -Datenformaten austauschen (-Schnittstelle).

Datenbank-Schnittstelle

Über die Datenbank-Schnittstelle können -Netzdaten zum Beispiel aus einer Access-Datenbank importiert oder in eine Access-Datenbank exportiert werden (Datenbank-Schnittstelle).

Attribut-Schnittstelle

Mit Hilfe der Attributschnittstelle können die Attributwerte der -Netzobjekte (zum Beispiel Knotennamen oder Streckenbelastungen) mit anderen Programmen oder anderen -Netzen ausgetauscht werden. Der Datenaustausch kann über Attributsdateien oder über die Windows-Zwischenablage erfolgen (Attribut-Schnittstelle).

Shapefile-Schnittstelle

Shapedateien sind geeignet für den Austausch von Daten mit Geographischen Informationssystemen (GIS). Über die Shapefile-Schnittstelle können Shapedateien direkt in eingelesen und aus geschrieben werden (Shapefiles-Schnittstelle).

Die Import-Funktion ist immer verfügbar, der Export ist nur mit dem Zusatzmodul GIS-Schnittstelle Shape möglich.

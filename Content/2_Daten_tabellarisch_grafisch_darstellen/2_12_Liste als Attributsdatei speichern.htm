Liste als Attributsdatei speichern

Alle Attributsdateien sind Textdateien mit der Endung *.att. In einer Attributsdatei speichern Sie die im aktuellen Listenfenster ausgegebenen Daten, d. h. die Attributwerte der Netzobjekte.

Stellen Sie sicher, dass das gewünschte Listenfenster geöffnet ist.

Nehmen Sie die gewünschten Listeneinstellungen vor.

Klicken Sie in der Symbolleiste des Listenfensters auf das Symbol Liste als Attributsdatei speichern.

Das Fenster Attribute speichern öffnet sich.

Geben Sie einen Dateinamen und bei Bedarf den Pfad ein.

Attributsdateien werden standardmäßig in das Verzeichnis geschrieben, das in den Projektverzeichnissen angegeben ist (Projektverzeichnisdatei öffnen und speichern).

Die Dateierweiterung *.att gilt für alle Attributsdateien, unabhängig von der Listenart, mit der sie erstellt wurden. Sie sollten die Datei daher entsprechend benennen.

Klicken Sie auf die Schaltfläche Speichern.

Das Fenster Attribute in Datei schreiben (<Netzobjekttyp>) öffnet sich.

Schlüsselfelder

Schlüsselfelder sind Attributen zugeordnet, die zur eindeutigen Identifikation der Netzobjekte dienen (siehe „Schlüsselfeldattribute“ auf Seite 1461).

Die Attribute werden gemäß den Einstellungen in eine Datei *.att gespeichert.

Schlüsselfeldattribute

Schlüsselfeldattribute identifizieren Netzobjekte eindeutig und werden beim Speichern automatisch als erste Spalte (oder Spalten) in eine Attributsdatei eingefügt.

In der Tabelle sehen Sie, welche Attribute als Schlüsselfeldattribute für das entsprechende Netzobjekt gespeichert werden, wenn Sie die jeweilige Auswahl treffen.

Für alle nicht aufgeführten Netzobjekttypen gibt es nur die jeweilige Standard-Kategorie von Schlüsselfeldern, die Sie alternativ zur Option wie die Liste wählen können und die entsprechend definiert ist, beispielsweise FplFahrtNr + Nr für Fahrplanfahrtabschnitte.

Wenn Sie die Option wie die Liste auswählen, können Sie die erzeugte Datei *.att möglicherweise nicht wieder einlesen, denn es werden nur die Attribute gespeichert, die in der aktuellen Liste ausgegeben sind. Schlüsselfelder, die nicht im Listenfenster ausgegeben sind, werden nicht gespeichert. Ohne Schlüsselfelder ist eine Datei *.att nicht einlesbar und prüft nicht, ob die aktuelle Liste Schlüsselfelder enthält.

Schlüsselfelder können beim Einlesen von Attributen nicht überschrieben werden.

Wenn Ihre Liste Aliase statt der herkömmlichen Attributsbezeichnungen enthält, werden diese in der Attributsdatei gespeichert (Alias für Attributnamen festlegen).

Wenn Sie diese Datei wieder einlesen, prüft zunächst, ob eine Spaltenbezeichnung einer Attributs-ID entspricht. Ist dies der Fall, wird diese vorgeschlagen. Ist dies nicht der Fall, prüft , ob es sich um einen gültigen Alias handelt.

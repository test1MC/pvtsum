RailML®-Schnittstelle

Diese Funktion ist nur verfügbar, wenn Sie die Zusatzmodule RailML Import und RailML Export aktiviert haben (Zusatzmodule ein- und ausschalten).

Über die railML®-Schnittstelle können Sie Fahrplandaten im railML®-Format in einlesen. Der Vorgang entspricht einem additiven Netzlesen (Netzdaten additiv einlesen), d. h. die Daten werden in ein bestehendes Netz eingelesen. Außerdem können Sie -Daten nach railML® exportieren (-Daten nach railML® exportieren).

RailML® ist ein Datenformat, das zwischen verschiedenen Mitwirkenden als gemeinsames universelles Format vereinbart wurde. Es liegt in Form eines XML-Schemas mit inhaltlichen Beschreibungen vor. Details finden Sie unter http://www.railml.org. In kann dieses Format importiert werden. Für den Import legen Sie die einzulesende Datei sowie weitere Importparameter fest. Die Einstellungen können Sie in ein railML®-Projekt *.puti speichern und später wiederverwenden (Einstellungen für den railML®-Import speichern und Einstellungen für den railML®-Import einlesen).

Der Import konzentriert sich auf Fahrplandaten und liest andere Schemata nur, soweit dies für den Import der Fahrplandaten notwendig ist (Hinweise zu den importierten railML®-Daten). In entstehen so Objekte der Linienhierarchie.

RailML®-Daten importieren

Sie können Dateien vom Typ *.xml importieren, die das railML®-Schema in der Version 2.0 oder 2.1 realisieren. Für den Import wählen Sie nur eine Datei aus, die jedoch auf weitere über ein „include”-Statement verweisen kann. Diese müssen im gleichen Verzeichnis liegen wie die ausgewählte Datei.

Wählen Sie im Menü Datei den Eintrag Importieren > RailML.

Das Fenster Import von RailML öffnet sich.

Wählen Sie das Register Grundeinstellungen.

Klicken Sie neben Import aus Datei auf die Schaltfläche .

Das Fenster RailML-Datei öffnen öffnet sich.

Wählen Sie die gewünschte Datei aus.

Klicken Sie auf die Schaltfläche Öffnen.

Der Dateiname und der Pfad werden im Eingabefeld übernommen.

Nehmen Sie weitere gewünschte Einstellungen vor.

Element

Beschreibung

Richtungen

Die Richtungen der in entstehenden Linienrouten werden aus der Zugnummer bestimmt. Ist keine Zugnummer angegeben, wird die Hinrichtung angenommen.

Ist eine Nummer angegeben, können Sie über die Option festlegen, ob gerade Nummern als Hinrichtungen angelegt werden und ungerade als Rückrichtungen oder umgekehrt.

Linienroute

Als Laufweginformation liegt eine Folge von Haltepunkten in vor. Hinzu kommen Informationen zu Ankunfts- und Abfahrtszeiten sowie Ein- und Ausstiegen. Für jeden Haltepunkt mit Zeiten wird in ein Routenpunkt auf der Linienroute erzeugt. Über folgende Optionen können Sie weitere Routenpunkte erzeugen.

Zusätzliche Routenpunkte erzeugen für alle explizit angegebenen Haltepunkte

Ist die Option ausgewählt, werden Routenpunkte für alle Haltepunkte erzeugt, die in der railML®-Datei enthalten sind und im Zielnetz gefunden wurden, auch jene ohne Zeiten oder Längen.

Zusätzliche Routenpunkte erzeugen für alle Haltepunkte mit Nach-Längenangabe

Ist die Option ausgewählt, werden Routenpunkte auch für alle Haltepunkte erzeugt, die in der railML®-Datei mit einer Nach-Längenangabe (‚distance‘) versehen sind.

Keine zusätzlichen Routenpunkte

Ist die Option ausgewählt, werden nur Routenpunkte für Haltepunkte mit Zeiten erzeugt.

Bei Wechsel der railML-Kategorie eigene Fahrt je Kategorie anlegen

Ist die Option ausgewählt, legt beim Wechsel der railML®-Kategorie eine Fahrplanfahrt für jede Kategorie an.

Hinweis

So werden beispielsweise Leerfahrten zu separaten Fahrplanfahrten.

Zugverbände koppeln (unter Umständen werden Fahrplanfahrtabschnitte zusammen gefasst)

Ist die Option nicht ausgewählt, werden keine Kopplungen eingelesen.

Keine Warnungen während des Imports anzeigen

Mit dieser Option können Sie die Anzeige von Warnungen unterdrücken.

Wählen Sie das Register Abbildung auf Zielnetz.

Element

Beschreibung

Sekundäres Korrespondenz-Attribut für Haltepunkte (Gleisinformation) verwenden

Ist die Option ausgewählt, können Sie ein Attribut auswählen um Gleisinformationen innerhalb einer Betriebsstelle (‚trackinfo‘) in einem -Attribut zu speichern.

Hinweis

Die Suche nach Entsprechungen im Zielnetz kann auf unterschiedlichen Detaillierungsebenen erfolgen. Sie können entweder nur ein primäres -Attribut auswählen, um Betriebsstellen auf jeweils einen Haltepunkt abzubilden, oder zusätzlich ein sekundäres -Attribut für Gleisinformationen innerhalb der Betriebsstelle auswählen. Darüber hinaus können Sie Streckeninformationen abgleichen.

Streckeninformation verwenden

Ist die Option ausgewählt, können Sie Korrespondenz-Attribute für Streckeninformationen auswählen.

Ist die Option nicht ausgewählt, werden Linienrouten über die Kurzwegsuche ermittelt.

Im primären Haltepunkt-Korrespondenz-Attribut Leerzeichen und Unterstrich identifizieren

Ist die Option ausgewählt, werden in der railML®-Datei und in Leerzeichen durch Unterstriche ersetzt, bevor nach Entsprechungen gesucht wird.

Hinweise

Diese Option ist sinnvoll, wenn in der einzulesenden Datei Leerzeichen statt Unterstriche verwendet werden.

Da in die benötigten Codes eindeutig sein müssen, scheitert ein Import mit dieser Option, falls sich zwei Haltepunkte im Identifikationsattribut nur durch dieses Zeichen unterscheiden.

Abbildung von Operation Control Points (OCPs) auf Haltepunkte

Hier ordnen Sie den Betriebsstellen („ocp”) in railML®-Haltepunktattribute zu.

Hinweise

Die Auswahl der railML®-Elemente hängt vom Inhalt der einzulesenden railML®-Datei ab.

Im Zielnetz können Sie benutzerdefinierte Haltepunktattribute mit passenden Namen anlegen und darin die Informationen speichern.

Primäres Korrespondenz-Attribut (laufwegunabhängig)

Auswahl eines laufwegunabhängigen railML®-Elements, dessen Information in einem -Haltepunktattribut gespeichert werden soll

Sekundäres Korrespondenz-Attribut (Gleisinformation, laufwegabhängig)

Auswahl eines laufwegabhängigen railML®-Elements, dessen Information in einem -Haltepunktattribut gespeichert werden soll

Hinweise

Die Auswahl ist nur möglich, wenn Sie die Option Sekundäres Korrespondenz-Attribut für Haltepunkte (Gleisinformation) verwenden ausgewählt haben.

Hier handelt es sich um Gleisinformationen innerhalb einer Betriebsstelle.

Korrespondenz-Attribute für Streckeninformation

Ist die Option Streckeninformation verwenden ausgewählt, können Sie folgenden railML®-Attributen -Streckenattribute zuordnen.

Für railML-Attribut ‚section‘

Hier ordnen Sie das Attribut ‚section‘ einem Streckenattribut zu, das die Streckennummer übernimmt.

Für railML-Attribut ‚trackinfo‘

Hier ordnen Sie das Attribut ‚trackinfo‘ einem Streckenattribut zu, das die Gleisinformation auf der Strecke übernimmt.

Hinweis

Wenn Sie ein benutzerdefiniertes Attribut für die Richtung einfügen, können Sie den Strecken in die gewünschte Richtung zuordnen.

Fehlende oder nicht eindeutige Informationen für den Verlauf von Linienrouten werden beim railML®-Import in stets über Kurzwegsuchen ermittelt.

Wählen Sie das Register Verkehrssysteme.

Die Einstellungen im Register Verkehrssysteme sind nur zugänglich, wenn Sie im Register Grundeinstellungen eine railML®-Datei ausgewählt haben.

Zeitverschiebungen

Zeitverschiebung im Zielnetz in Bezug auf UTC

Hier können Sie die Ankunfts- und Abfahrtszeiten von Fahrten in in Bezug auf die koordinierte Weltzeit verschieben. Eine Verschiebung um eine Stunde entspricht der MEZ.

Wählen Sie das Register Namen.

In diesem Register legen Sie fest, aus welchen Daten die Namen der Linien und Fahrplanfahrten in gebildet werden. Jeweils die linke Liste enthält die Daten, aus denen sich die Namen in der gegebenen Reihenfolge zusammensetzen. Die rechte Liste enthält alle Daten, die für die Namensbildung zur Verfügung stehen.

Gibt es in eine Linie mit den angegebenen Namensbestandteilen, wird diese verwendet, falls das Verkehrssystem passt. Gibt es keine Linie mit dem angegebenen Namen, wird eine neue angelegt. Gibt es eine namensgleiche Linie, aber mit einem anderen als dem verlangten Verkehrssystem, gelingt der Import für diese Linie nicht. Diese Situation kann vermieden werden, wenn der Verkehrssystem-Code als Bestandteil des Liniennamens ausgewählt wird.

Wählen Sie im Bereich Linienname in der linken Liste den Eintrag, über dem Sie eine weitere Namenskomponente hinzufügen möchten.

Wählen Sie im Bereich Linienname in der rechten Liste den Eintrag, den Sie zum Liniennamen hinzufügen möchten.

Klicken Sie auf das Symbol .

Der Eintrag wird in die linke Liste über dem aktuell ausgewählten Eintrag eingefügt.

Geben Sie bei Bedarf im Bereich Linienname in das Eingabefeld Trennzeichen ein Zeichen zur Trennung mehrerer Namenskomponenten ein.

Das Semikolon ist als Trennzeichen nicht erlaubt.

Fügen Sie bei Bedarf weitere Einträge hinzu.

Sie können einen Eintrag aus der linken Liste löschen, indem Sie den gewünschten Eintrag auswählen und auf die Schaltfläche klicken.

Legen Sie auf die gleiche Weise im Bereich Fahrtname fest, aus welchen Daten die Fahrtnamen gebildet werden sollen.

Geben Sie in die Eingabefelder Fester Namensbestandteil für Linie und Fester Namensbestandteil für Fahrt jeweils die gewünschte Zeichenfolge ein.

Diese Zeichenfolge steht für die Namensbildung als Eintrag Fester Namensbestandteil zur Verfügung.

Nehmen Sie im Bereich Linienroutenname die gewünschte Einstellung vor.

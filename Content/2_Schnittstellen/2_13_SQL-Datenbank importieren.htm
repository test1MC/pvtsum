SQL-Datenbank importieren

Sie können vollständige Netze aus einer bestehenden SQL-Datenbank oder ausgewählte Netztabellen additiv zum aktuellen Netz einlesen.

Die Datenbank muss mindestens die für die Einlesbarkeit notwendige Tabelle $Version enthalten.

Wählen Sie im Menü Datei den Eintrag Importieren > Datenbank.

Das Fenster Datenbank auswählen öffnet sich.

Wählen Sie den Eintrag MS SQL Server.

Das Fenster Datenverknüpfungseigenschaften öffnet sich.

Wählen Sie das Register Provider.

Wählen Sie den geeigneten Provider für SQL-Server, beispielsweise Microsoft OLE DB Provider for SQL Server.

Wählen Sie das Register Verbindung.

 

Alternativ können Sie im Register Provider auf die Schaltfläche Weiter klicken.

Geben Sie den Namen des Servers ein, auf dem sich die SQL-Datenbank befindet.

 

Alternativ können Sie den Namen in der Auswahlliste wählen.

Geben Sie den Benutzernamen und Passwort für den Zugriff auf den SQL-Server ein.

Wählen Sie die Option Speichern des Kennworts zulassen.

Wählen Sie in der Auswahlliste die gewünschte Datenbank auf dem SQL-Server.

 

Sie können prüfen, ob die Verbindung zum SQL-Server besteht und der Export in die Datenbank erfolgen kann, indem Sie auf die Schaltfläche Verbindung testen klicken.

Das Fenster Netz lesen öffnet sich.

Nehmen Sie die gewünschten Einstellungen vor (Netzdatei öffnen und Netzdaten additiv einlesen).

Der Import der Netzdaten aus der SQL-Datenbank wird gestartet. Wenn Sie die Option Netzdatei additiv lesen ausgewählt haben, öffnet sich zusätzlich das Fenster Netzdaten hinzulesen (Netzdaten additiv einlesen). Wenn die eingelesenen Netzdaten unvollständige Linien- oder Systemrouten enthalten, d. h. mindestens eine Strecke oder ein Haltepunkt im Routenweg fehlt oder für den Routenweg gesperrt ist, öffnet sich zusätzlich das Fenster Netz lesen (Linienroutenweg / Systemroutenweg ergänzen) (Netzdatei mit unvollständigen Linien- oder Systemrouten einlesen).

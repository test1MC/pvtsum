EMME/2-Daten importieren

Wählen Sie im Menü Datei den Eintrag Importieren > Emme.

Das Fenster Import von EMME/2 öffnet sich.

Wählen Sie das Register Dateien.

Klicken Sie je EMME/2-Datendateiart, für die Sie Daten importieren möchten, auf die Schaltfläche .

Das Fenster <Dateiart>öffnen öffnet sich.

Wählen Sie die gewünschte Datei aus.

Klicken Sie auf die Schaltfläche Öffnen.

Der Dateiname und der Pfad werden im jeweiligen Eingabefeld Pfad übernommen.

Sie können die Zuordnung einer zu importierenden Datei löschen, indem Sie auf die jeweilige Schaltfläche Löschen klicken.

Wählen Sie das Register Einheiten.

Element

Beschreibung

Längen / Geschwindigkeiten

Über die Auswahlliste können Sie die Einheit auswählen, die in den EMME/2-Daten für Längen und Geschwindigkeiten verwendet wird.

  • Kilometer
  • Meilen

Faktor

Faktor, mit dem die Werte aus EMME/2 in die in verwendeten Einheiten umgerechnet werden.

Koordinaten

Über die Auswahlliste können Sie die Einheit auswählen, die in den EMME/2-Daten für Koordinaten verwendet wird:

  • Fuß
  • Kilometer
  • Meilen

Faktor

Faktor, mit dem die Werte aus EMME/2 in die in verwendeten Einheiten umgerechnet werden.

Hinweis

Die Koordinaten werden in unverändert übernommen. Der Netzmaßstab wird angepasst, um das Verhältnis der Koordinaten und Längen korrekt wiederzugeben (Koordinatensystem und Maßstab ändern).

Wählen Sie das Register Strecken.

Element

Beschreibung

Kapazität IV

Über die Auswahlliste können Sie wählen, ob Strecken mit einer konstanten IV-Kapazität importiert werden sollen oder für die IV-Kapazität je Strecke der Wert eines EMME/2-benutzerdefinierten Attributs übernommen werden soll.

konstant

Ist die Option ausgewählt, geben Sie den Wert der IV-Kapazität in das Eingabefeld ein.

v0 IV

Über die Auswahlliste können Sie ein EMME/2-benutzerdefiniertes Attribut wählen, dessen Wert je Strecke für die Geschwindigkeit im unbelasteten Netz übernommen werden soll.

Für Strecken interpretiert den Parameter a in EMME/2-Dateien und fügt automatisch beide Richtungen einer Strecke mit gleichen Attributwerten ein.

Wählen Sie das Register Verkehrssysteme.

Die Einstellungen im Register Verkehrssysteme sind nur zugänglich, wenn Sie eine EMME/2 Modes-Datei für den Import ausgewählt haben.

Ist eine EMME/2 Modes-Datei für den Import ausgewählt, werden EMME/2-Verkehrssysteme den -Verkehrssystemtypen automatisch gemäß folgender Regel zugeordnet:

  • EMME/2 mode type 1 = -Verkehrssystemtyp IV
  • EMME/2 mode type 2 = -Verkehrssystemtyp ÖV
  • EMME/2 mode type 3 (transit walk links) = -Verkehrssystemtyp ÖVFuß
  • EMME/2 mode type 4 (walk links) = -Verkehrssystemtyp IV

Ändern Sie bei Bedarf die Zuordnung von EMME/2-Verkehrssystemen zu -Verkehrssystemtypen über die jeweilige Auswahlliste Visum VSysTyp.

Für jedes IV-Verkehrssystem wird automatisch je ein Modus und je ein Nachfragesegment erstellt, für sämtliche Verkehrssysteme des Typs ÖV, ÖVFuß und ÖVZusatz wird nur ein Modus und ein Nachfragesegment erstellt.

Falls kein Verkehrssystem ÖV-Fuß importiert wird, wird ein Standard-Verkehrssystem ÖVFuß angelegt, das auf allen Anbindungen, jedoch auf keinen Strecken zugelassen ist.

Wählen Sie das Register Nachfragematrizen.

Die Einstellungen im Register Nachfragematrizen sind nur zugänglich, wenn Sie eine EMME/2 Modes-Datei für den Import ausgewählt haben.

Klicken Sie je Nachfragesegment, für das Sie eine EMME/2-Nachfragematrix importieren möchten, auf die Schaltfläche .

Das Fenster öffnen öffnet sich.

Wählen Sie die gewünschte Datei aus.

Klicken Sie auf die Schaltfläche Öffnen.

Der Dateiname und der Pfad werden im jeweiligen Eingabefeld Pfad übernommen.

 

Sie können die Zuordnung einer Matrix zu einem Nachfragesegment löschen, indem Sie auf die jeweilige Schaltfläche Löschen klicken.

Alternativ können Sie Nachfragematrizen, die in einem Fremdformat vorliegen, direkt in oder im Matrixeditor einlesen (Matrizen verwalten).

Zusätzlich zur Nachfragematrix wird für das ÖV-Nachfragesegment eine Standardganglinie erzeugt.

Wählen Sie das Register TTF.

Die Einstellungen im Register TTF sind nur zugänglich, wenn Sie eine EMME/2 Transit Lines-Datei für den Import ausgewählt haben.

Element

Beschreibung

TTF-Index

Index-Nummer der zu importierenden TTF-Funktion (Transit Time Function) zur Berechnung der Geschwindigkeit von Linienrouten. Sie wird aus der ausgewählten EMME/2 Transit Lines-Datei eingelesen.

Streckenattribut

Sie können festlegen, ob die Geschwindigkeit der Linienroute konstant sein soll oder ob sie aus einem Streckenattribut und einem Faktor berechnet werden soll.

Über die Schaltfläche können Sie in einem separaten Fenster entweder die konstante Geschwindigkeit oder ein beliebiges Streckenattribut auswählen.

Faktor / Geschw.

Ist als Streckenattribut die konstante Geschwindigkeit ausgewählt, geben Sie den Wert der Geschwindigkeit ein.

Ist ein beliebiges Streckenattribut ausgewählt, geben Sie einen Faktor ein, mit dem der Attributwert multipliziert wird.

Für jede importierte EMME/2-Haltestelle einer EMME/2-Linie legt einen Knoten, eine Haltestelle, einen Haltepunkt und einen Haltestellenbereich an, wobei der Haltestellenbereich dem Knoten zugeordnet ist.

Da EMME/2 nicht zwischen Haltestellen und Knoten unterscheidet (Haltestellen, Bezirke, Knoten = „centroids“), wird jeder Centroid im Verlauf einer Linienroute als Haltestelle interpretiert, sofern keine explizite Angabe in EMME/2 vorliegt, dass die Linienroute an dieser Stelle nicht hält.

Bei einem unvollständigen Verlauf in EMME/2 können Sie den entsprechenden Linienroutenverlauf wie beim Netzlesen ergänzen (Netzdatei mit unvollständigen Linien- oder Systemrouten einlesen).

Wählen Sie das Register Allgemein.

Element

Beschreibung

Für jede EMME/2-Anbindung eigenen Knoten erzeugen und über zusätzliche Strecke mit dem Netz verbinden

Ist die Option ausgewählt, wird für jeden EMME/2-Centroid ein Bezirk und ein Knoten erzeugt, sowie eine Anbindung, die beide verbindet. Für alle EMME/2-Strecken von/zu dem EMME/2-Centroid werden Strecken vom/zum neuen Knoten erzeugt.

Ist die Option nicht ausgewählt, werden EMME/2-Centroids lediglich als Bezirke eingelesen. erzeugt einen Bezirk und je EMME/2-Strecke eine Anbindung.

Linien in Hin- und Rückrichtung aufteilen

Ist die Option ausgewählt, werden auf der Basis der EMME/2-Liniendaten jeweils Hin- und Rückrichtung einer Linienroute erzeugt.

Hinweis

Anhand der in EMME/2 angegebenen Aufenthaltszeit („lay“ = layover) wird festgelegt, wo in eine Linienroute beginnt oder endet. Wird ein „layover“ innerhalb eines EMME/2-Linienverlaufs angeben, dann werden in Hin- und Rückrichtung erzeugt, also zwei Linienrouten. Wenn „layover“ am Ende des Verlaufs steht oder nicht definiert ist, erzeugt nur eine Richtung und Linienroute.

Ist die Option nicht aktiv, wird stets nur eine Richtung erzeugt.

Die ausgewählten EMME/2-Daten werden importiert.

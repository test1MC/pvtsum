Abstrahiertes Netzmodell speichern

Für den Export nach können Sie -Netzdaten als abstrahiertes Netzmodell in einer Datei *.anm speichern. Routen und Matrizen können Sie in eine Routendatei *.anmRoutes speichern. Beide Dateiarten werden im Format *.xml gespeichert (Kurze Einführung zum Format *.xml).

Wählen Sie im Menü Datei den Eintrag Exportieren > Vissim (ANM).

Das Fenster ANM – Export Parameter öffnet sich.

Element

Beschreibung

ANM-Datei

Netzdaten

Ist die Option ausgewählt, können Sie die Netzdaten des aktuellen Netzes in eine Datei *.anm speichern. Geben Sie einen Dateinamen und bei Bedarf den entsprechenden Pfad in das Eingabefeld ein oder klicken Sie auf die Schaltfläche Durchsuchen.

ANMRoutes-Datei

Routen

Ist die Option ausgewählt, werden Routeninformationen in die Routendatei *.anmRoutes gespeichert. Geben Sie einen Dateinamen und bei Bedarf den entsprechenden Pfad in das Eingabefeld ein oder klicken Sie auf die Schaltfläche Durchsuchen.

Hinweis

Beim Import in Vissim können Sie wählen, ob die Informationen als statische oder als dynamische Wegeinformationen eingelesen werden.

Matrizen

Ist die Option ausgewählt, werden die Nachfragematrizen des aktuellen Netzes in eine ANM-Routendatei *.anmRoutes gespeichert. Geben Sie bei Bedarf einen Dateinamen und den entsprechenden Pfad in das Eingabefeld ein oder klicken Sie auf die Schaltfläche Durchsuchen.

Wählen Sie das Register Fahrzeugkategorien.

Ordnen Sie über die Auswahllisten jedes -Verkehrssystem, das Sie exportieren möchten, einer Vissim-Fahrzeugkategorie zu.

Verkehrssysteme des Typs ÖVFuß werden nicht nach Vissim exportiert und sind daher nicht in der Liste enthalten.

Wenn Sie ein -Verkehrssystem nicht nach Vissim exportieren möchten, wählen Sie in der jeweiligen Auswahlliste den Eintrag Nicht exportieren.

Wählen Sie das Register Fahrverhalten.

Wählen Sie über die Auswahllisten für jeden -Streckentyp ein Vissim-Standard-Fahrverhalten.

Alternativ können Sie das Fahrverhalten für mehrere Streckentypen in einem separaten Fenster gleichzeitig setzen, indem Sie die Taste Strg drücken, diese gedrückt halten, auf die gewünschten Zeilen mit der linken Maustaste klicken und auf die Schaltfläche Mehrere klicken.

Wählen Sie das Register ÖV.

Nehmen Sie im Bereich Vissim - Haltestellen die gewünschten Einstellungen vor.

Element

Beschreibung

Länge der Haltestelle aus Haltepunktattribut

Über die Schaltfläche wählen Sie ein Attribut, aus dem die Länge der Haltestellen in Vissim bestimmt werden soll.

Tipp

Sie können beispielsweise ein benutzerdefiniertes Attribut für Haltepunkte anlegen und mit Werten befüllen und dieses Attribut auswählen.

Als Bucht, falls Haltepunktattribut wahr, bzw. ungleich 0

Über die Schaltfläche wählen Sie ein Haltepunktattribut aus, um zu bestimmen, ob Haltepunkte als Buchten oder Kaps exportiert werden.

Tipp

Sie können beispielsweise allen Haltepunkten, die Sie als Buchten exportieren möchten, einen ZWert von 1 zuweisen und allen Haltepunkten, die Sie als Kap exportieren möchten, eine 0. Wenn Sie nun das Attribut ZWert auswählen, werden die Haltepunkte wie gewünscht exportiert.

Knotenhaltepunkt vor dem Knoten, falls Haltepunktattribut wahr, bzw. ungleich 0

Über die Schaltfläche wählen Sie ein Haltepunktattribut aus, welches bestimmt, ob der Haltepunkt vor oder nach dem Knoten liegt.

Hinweise

Dies betrifft nur Knotenhaltepunkte.

Falls eine Linienroute in einem Knoten endet, wird der letzte Halt immer vor den Knoten gelegt.

Falls kein Attribut ausgewählt ist, liegen die Haltepunkte hinter dem Knoten.

Nehmen Sie im Bereich ÖV-Telegramme die gewünschten Einstellungen vor.

Element

Beschreibung

Linie sendet ÖV-Telegramme

Über die Schaltfläche wählen Sie ein Attribut der Fahrplanfahrt, das festlegt, ob die Fahrplanfahrt (die in Vissim einer „Linie“ entspricht) ÖV-Telegrammdaten exportiert. Der Wert wird in Vissim im Fenster ÖV-Telegramm im Kontrollkästchen Linie sendet ÖV-Telegramme angezeigt.

Ist der Attributwert in null, ist das Kontrollkästchen inaktiv.

Linie

Attribut der Fahrplanfahrt. Die Werte werden in Vissim im Fenster ÖV-Telegramm im Feld Linie angezeigt.

Route

Attribut der Fahrplanfahrt. Die Werte werden in Vissim im Fenster ÖV-Telegramm im Feld Route angezeigt.

Priorität

Attribut der Fahrplanfahrt. Die Werte werden in Vissim im Fenster ÖV-Telegramm im Feld Priorität angezeigt.

Zuglänge

Attribut der Fahrplanfahrt. Die Werte werden in Vissim im Fenster ÖV-Telegramm im Feld Zuglänge angezeigt.

Richtung von Hand

Attribut der Fahrplanfahrt. Die Werte werden in Vissim im Fenster ÖV-Telegramm im Feld Richtung von Hand angezeigt.

Wählen Sie das Register Andere Einstellungen.

Element

Beschreibung

Simulationszeitintervall

von / bis

Geben Sie in die Eingabefelder von und bis Anfang und Ende eines Zeitintervalls für die Vissim-Simulationsdauer im Uhrzeitformat (hh:mm:ss) ein.

Hinweise

Anfang und Ende des Zeitintervalls dürfen an zwei aufeinanderfolgenden Tagen liegen, solange die Dauer nicht 24 Stunden überschreitet.

Bei Verwendung eines Kalenders müssen Sie zusätzlich einen Wochentag oder ein Datum wählen (Kalender auswählen).

Anz. Nachkommastellen für Koordinaten

Maximal

Ist die Option ausgewählt, werden die Koordinaten mit der maximal vorhandenen Genauigkeit verwendet.

Reduziert (4)

Ist die Option ausgewählt, werden die Koordinaten auf die vierte Nachkommastelle gerundet.

Hinweis

Wir empfehlen diese Einstellung, wenn Sie die Datei in einer -Version vor 14 erstellt haben und diese in adaptiv einlesen möchten.

LSA-Signalprogramm- oder Tagesplannummer für VISSIG-Steuerungen

Aus Attribut

Ist die Option ausgewählt, wählen Sie über die Schaltfläche das gewünschte LSA-Attribut aus. Standardmäßig ist die Signalprogrammnummer ausgewählt. Diese Einstellung bewirkt, dass das derzeit gewählte Vissig-Signalprogramm exportiert wird. Dies ist sinnvoll, wenn es je LSA nur ein Signalprogramm gibt, oder wenn Sie an allen LSAs das für den Export relevante Signalprogramm ausgewählt haben. Sie können Signalprogramme manuell auswählen (Attribute und Optionen von Signalanlagen und Signalprogramm in manuell wechseln oder ausschalten) oder während des Verfahrensablaufs automatisch setzen (Verfahren in Verfahrensablauf einfügen).

Statt der Signalprogrammnummer können Sie beispielsweise ein benutzerdefiniertes Attribut wählen, das die Tagesplannummer enthält.

Eindeutige Nummer

Ist die Option ausgewählt, geben Sie in das Eingabefeld eine feste Nummer ein. Dies ist sinnvoll, wenn Sie für das gewählte Zeitintervall für alle LSAs die gleiche Tagesplannummer verwenden (Beispiel: Montag 101, Dienstag 102 usw.).

Attribut zur Belegung der Koordinierungsgruppe

Über die Schaltfläche können Sie ein Attribut der Lichtsignalanlagen für die Koordinierungsgruppe auswählen.

Hinweis

Das Attribut wird ausschließlich beim Import in verwendet.

Einstellungen für andere Objekte

Attribut zur Belegung des Langsamfahrbereichs bei Fahrstreifenabbiegern

Über die Schaltfläche können Sie ein Fahrstreifenabbieger-Attribut auswählen, um eine Geschwindigkeit anzugeben.

Hinweis

Diese Geschwindigkeit wird beim Import in Vissim zur Erzeugung des Langsamfahrbereichs verwendet. Wird kein Attribut ausgewählt, verwendet das Programm Standardwerte.

Attribut zur Belegung der Sättigungsverkehrsstärke bei Strecken

Über die Schaltfläche können Sie ein Strecken-Attribut auswählen.

Hinweis

Das Attribut wird ausschließlich beim Import in verwendet.

Attribut zur Belegung der Sättigungsverkehrsstärke bei Abbiegern

Über die Schaltfläche können Sie ein Abbieger-Attribut auswählen.

Hinweis

Das Attribut wird ausschließlich beim Import in verwendet.

Es gibt in Knoten, für die Abbieger zugelassen sind, die aber keine Fahrstreifendefinitionen haben. Wenn Sie nach dem ANM-Export aus auch Verbinder für diese Knoten in Vissim haben möchten, deaktivieren Sie das (Ober-)Knotenattribut Fahrstreifendefinition verwenden. Dann werden für diese Knoten nicht die detaillierten Geometriedaten (d.h. hier die Fahrstreifenabbieger) verwendet, sondern die Informationen auf Abbiegerebene.

Das abstrahierte Netzmodell wird gespeichert.

Beim ANM-Export eines -Netzes mit einer Projektion wird diese Projektion auch in die ANM-Datei geschrieben, so dass die geografische Lage der Daten in bekannt ist. Die Anzeige von Hintergrundkarten ist dadurch möglich. Um diese Funktion nutzen zu können sind die -Versionen ab 5.40-11 oder 6.00-09 erforderlich.
Beim ANM-Export wird eine Lichtsignalanlagen exportiert, wenn sie mindestens einem signalisierten Knoten oder Oberknoten zugewiesen ist.

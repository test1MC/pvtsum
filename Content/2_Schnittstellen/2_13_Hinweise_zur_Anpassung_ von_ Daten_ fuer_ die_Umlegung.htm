Hinweise zur Anpassung von Daten für die Umlegung

Ein Hauptgrund für den Import von -Netzen in ist die Vielfältigkeit der Kostenfunktionen, die für die Umlegung in auswählbar sind. Dadurch haben Sie mehr Möglichkeiten die Wege, die in der in der Umlegung gefunden werden, zu beeinflussen. Beim Import in wird eine optimale Annäherung an die -Kosten erreicht.

verwendet die folgenden Kosten:

Netzobjekttyp in

Beschreibung

Anbindungen

3.6*conn.GetLengthDir()/10.0

Dauer zum Abfahren der Länge der Anbindung mit 10 km/h.

Strecken

3.6 * link.GetLengthPoly() / link.GetV0PrT()

Die Geschwindigkeit wird in explizit gesetzt. 100000h wird verwendet, wenn die Geschwindigkeit 0 ist, die Strecke keine Fahrstreifen hat oder für kein Verkehrssystem freigegeben ist.

Abbieger

0

100000h wird verwendet, wenn der Abbieger für kein Verkehrssystem freigegeben ist oder er keine Fahrstreifenabbieger hat.

Oberabbieger

100000h

In werden in den Umlegunsgverfahren die Werte t0 und tAkt verwendet. Die folgende Tabelle beschreibt die Vorgehensweise bei den einzelnen Netzobjekttypen.

Netzobjekttyp in

t0 in

Beim Import in

Anbindungen

t0 ist ein Eingabewert

t0 wird beim Import auf den in verwendeten Wert gesetzt.

Strecken

t0 wird aus Länge und v0 berechnet

Die Länge ist in immer 0. Beim Import wird sie auf die Polygonlänge gesetzt.

Die Streckenkapazität wird in Vistro schon immer gepflegt (Anzahl Fahrstreifen * 2000). Die Kapazitäten werden trotzdem noch einmal angepasst.

Abbieger

t0 ist ein Eingabewert

In können Sie eine CR-Funktion definieren oder sinnvollere Werte als 0 verwenden.

Oberabbieger

t0 ist ein Eingabewert

-Netze enthalten keine Oberknoten, daher ist dieser Wert irrelevant.

Belastung je Nachfragesegment oder additiv speichern

Wenn Sie mehrere ÖV-Nachfragesegmente gleichzeitig umlegen, werden die Ergebnisse standardmäßig für jedes Nachfragesegment getrennt gespeichert. Sie können die Ergebnisse auch additiv speichern. Sie legen dabei vorab fest, auf welches Nachfragesegment die Ergebnisse jeweils gespeichert werden. Beispielsweise speichern Sie die Ergebnisse der Nachfragesegmente A, B und C additiv auf das Nachfragesegment A, während die Ergebnisse des Nachfragesegments D weiterhin auf D gespeichert werden.

Bei großen Netzen sparen Sie Speicherplatz, wenn Sie die Ergebnisse additiv speichern.

Die Ergebnisse können nachträglich nicht mehr auf die einzelnen Nachfragesegmente differenziert werden.

Diese Funktion steht nur für ÖV-Nachfragesegmente zur Verfügung.

Sie können die Umlegung so durchführen, dass die Ergebnisse am Ende sowohl einzeln als auch aufsummiert zur Verfügung stehen. Legen Sie dafür ein zusätzliches Nachfragesegment an. Legen Sie fest, dass die Ergebnisse der anderen Nachfragesegmente auf das neue Nachfragesegment summiert werden und führen Sie die Umlegung für alle Nachfragesegmente durch. Die Summe aller Belastungen ist jetzt auf dem neuen Nachfragesegment gespeichert. Setzen Sie anschließend die Einstellungen wieder auf die Standardeinstellung zurück, sodass wieder jedes Ergebnis auf das jeweilige Nachfragesegment gespeichert wird. Rechnen Sie die Umlegung nochmals – diesmal jedoch ohne das neue Nachfragesegment mit einzubeziehen. Die Belastungen sind jetzt je Nachfragesegment gespeichert, während die Summe unverändert erhalten bleibt.

Wählen Sie im Menü Rechnen den Eintrag Allgemeine Verfahrenseinstellungen.

Das Fenster Allgemeine Verfahrenseinstellungen öffnet sich.

Wählen Sie im Navigator den Eintrag Belastungen.

Wählen Sie in der gewünschten Zeile in der Spalte Speichern bei Nachfragesegment in der Auswahlliste den gewünschten Eintrag.

Wenn schon Umlegungsergebnisse vorhanden sind, löschen Sie durch diese Änderung die Ergebnisse derjenigen Nachfragesegmente, für die Sie ein anderes Nachfragesegment zum Speichern der Ergebnisse wählen.

Die Änderungen werden übernommen. Wenn Sie eine Umlegung rechnen, werden die Ergebnisse auf das jeweils gewählte Nachfragesegment gespeichert.

Wenn Sie Umlegungsergebnisse additiv speichern möchten, müssen Sie bei der Umlegung alle Nachfragesegmente, auf die Ergebnisse additiv gespeichert werden sollen, mit umlegen.

Über die Schaltfläche Initialisieren können Sie die Einstellungen wieder auf den Standard zurücksetzen.
